출처: SRC-HARNESS-EDU-SNAPSHOT-20260718; 계산식: SELECT COUNT(*) FROM raw_signals WHERE domain='edu_consulting'; filtered_signals 및 refined_outputs 동일 domain 조인. 기간: 스냅샷 시각 2026-07-18. 한계: 로컬 DB와 Mac Mini 운영 DB의 시점 차이 가능.

04 / SIX-MONTH OPERATING PLAN

6개월 계획은
결과물의 연속성으로 관리한다

01–02 / 질문

  • 관심 질문 1개를 자기 말로 기록
  • 작은 실험 1개 종료
  • 첫 산출물 링크 보관

03–04 / 제작

  • 범위를 4주 단위로 축소
  • 실패·수정 로그 주 1회
  • 설명 가능한 데모 1개

05–06 / 검토

  • 결과물과 질문의 연결 설명
  • 다음 기술 장벽 1개 선택
  • 수업 구매 여부 재평가

관찰 KPI: question_saved, artifact_shipped, revision_logged, next_choice_explained. 네 지표 모두 내부 실행 로그로만 측정한다.

출처: SRC-PILOT-ROADMAP-20260718, 내부 운영 설계. 이는 교육 효과나 입시 결과의 예측 모델이 아니다. 각 KPI의 이벤트 정의는 후속 instrumentation에서 고정.

05 / FIVE-MINUTE DIAGNOSTIC

적성을 판정하지 말고
관찰 가능한 흔적을 점수화한다

아이가 ‘AI를 좋아한다’는 자기소개보다, 질문을 만들고 결과물을 수정했는지를 확인한다. 각 항목 0~2점, 총 8점.

질문0: 없음 · 1: 부모 도움 · 2: 자기 말로 문제 정의
제작0: 시작 전 · 1: 튜토리얼 복제 · 2: 4주 산출물
수정0: 기록 없음 · 1: 한 번 수정 · 2: 실패 원인과 변경 기록
지속0: 중단 · 1: 간헐적 · 2: 주간 로그 유지

판정: 6점 이상이면 프로젝트를 4주 연장 검토. 3~5점이면 범위를 축소하거나 한 영역만 보완. 0~2점이면 새 수업 구매보다 질문 재설계.

출처: SRC-PILOT-DIAGNOSTIC-20260718, 내부 가설형 도구. 점수와 임계값은 검증 전 가설이며 적성·성적·합격을 판정하지 않음.

06 / FAILURE ECONOMICS

실패를 없애는 것이 아니라
빨리 발견하는 비용 구조를 만든다

정량 손실은 아직 측정하지 않았다. 대신 구매 전에 중단 조건과 기록 항목을 고정해, 잘못된 선택이 길어지는 것을 막는다.

OVERBUYING

수업을 먼저 결제

중단 조건: 2주 후 질문·산출물 미기록. 조치: 추가 결제 보류, 범위 재설계.

OVERSCOPING

프로젝트가 너무 큼

중단 조건: 4주 안 데모 불가. 조치: 기능 하나만 남기고 재시작.

FALSE SIGNAL

완성도를 적성으로 착각

중단 조건: 설명·수정 로그 없음. 조치: 결과물보다 과정 리뷰.

So what: 부모가 구매 전에 ‘언제 멈출지’를 합의하면, 불안을 이유로 수업을 연속 구매하는 경로를 줄일 수 있다. 절감액은 측정 전 주장하지 않는다.

출처: SRC-PILOT-RISK-20260718, 내부 pre-mortem 가설. 비용·시간 절감 효과는 선결제 파일럿 3가구의 실제 로그 수집 후에만 계산.

07 / ACTION + METHOD

이번 주 행동과
근거의 위치를 함께 남긴다

아이에게 관심 질문 1개를 자기 말로 쓰게 한다
4주 안에 가능한 산출물 1개를 정한다
매주 질문·제작·수정·지속 로그를 저장한다
2주 후 추가 결제를 보류할지 판정한다
4주 후 8점 진단을 다시 계산한다
다음 선택의 이유와 반론을 한 줄로 쓴다

방법론: 로컬 raw_signals → filtered_signals → refined_outputs의 계층을 사용했다. 모든 관측 수치는 스냅샷 SQL로 재계산 가능하며, 가설은 HYPOTHESIS로 표시했다. 외부 발행 전 원문 URL·발행일·표본·추출 위치를 evidence ledger에 추가한다.

출처: SRC-HARNESS-EDU-SNAPSHOT-20260718; SRC-PILOT-MATRIX/ROADMAP/DIAGNOSTIC/RISK-20260718. 한계: 내부 운영 데이터는 학부모 모집단, 교육 효과, 입시 결과를 대표하지 않음. 면책: 투자·입시·성적·진로 결과를 보장하지 않음.
