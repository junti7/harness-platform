## 왜 음성 온보딩인가

**텍스트 폼의 한계:**
```
폼 질문: "AI를 어떤 방식으로 사용하고 있나요?"
→ 대부분 고객: 한두 줄 답변 → 맥락 부족 → 맞춤화 부정확

실제 고민은 이렇게 나온다:
"음... 우리 아이가요, 중2인데, 요즘 수학 숙제를 계속
 ChatGPT한테 물어봐요. 근데 저도 잘 모르거든요,
 어디까지가 괜찮고 어디부터가 문제인지. 학교에서
 뭐라고 하는지도 모르고... 그냥 불안한 거죠."
```

이 발화에 들어있는 정보:
- 세그먼트: 학부모 (Track B)
- 자녀: 중2
- 우려: ChatGPT 숙제 의존 + 경계 기준 불명확
- 감정 상태: 불안, 정보 부재
- 온보딩 진입점: "AI 의존 기준선 설정" + "학교 규칙 안내"

이 정보를 텍스트 폼으로 얻으려면 10개 질문이 필요하다. 음성으로 2분이면 충분하다.

---

## 시스템 아키텍처

```
[Step 1] 고객 음성 입력
   ↓ (2~5분 자유 발화)

[Step 2] STT 변환
   → Whisper local (무료, 오프라인) 또는 AssemblyAI API (유료, 빠름)
   ↓

[Step 3] 전사 텍스트 → Claude 분석
   → 세그먼트 분류 (학부모/직장인/대학생/노인)
   → 서브 트랙 분류 (Track A/B/C/D)
   → 핵심 고민 3가지 추출
   → 감정 상태 태깅 (공황/호기심/체념/분노/혼란)
   → 긴급도 판단 (즉각 처방 필요 / 탐색 단계)
   ↓

[Step 4] 온보딩 경로 결정
   → 맞춤 첫 번째 메시지 생성
   → 추가 확인이 필요한 질문 1~2개만 생성
   → 예상 서비스 플로우 제안

[Step 5] 대표/부대표 확인 (선택)
   → 자동 발송 (빠른 피드백) 또는 검토 후 발송
```

## STT 엔진 비교

| 항목 | Whisper (Local) | AssemblyAI | Deepgram |
|---|---|---|---|
| **비용** | 무료 (GPU 필요) | $0.37~$0.65/시간 | $0.0043/분 |
| **한국어 지원** | ✅ 우수 | ✅ 양호 | ⚠️ 제한적 |
| **속도** | 느림 (CPU 기준 2~5배 실시간) | 빠름 (실시간의 1/5) | 빠름 |
| **프라이버시** | ✅ 로컬 처리 (데이터 외부 전송 없음) | ⚠️ 클라우드 | ⚠️ 클라우드 |
| **정확도 (한국어)** | ★★★★☆ | ★★★★☆ | ★★★☆☆ |
| **세팅 복잡도** | 높음 | 낮음 (API) | 낮음 (API) |

**Phase 1 권고: Whisper local**
- 이유 1: 고객 음성 데이터는 개인 고민이 담겨 있어 프라이버시 민감
- 이유 2: 비용 무료 — Phase 1 규모에서 API 비용 절약
- 이유 3: Mac Mini M4에 설치 가능 (`pip install openai-whisper`)
- 단점: CPU만으로는 2분 음성 변환에 3~8분 소요 → 실시간 불가. "접수 후 30분 내 분석" 약속으로 해결.

**Phase 2 업그레이드: AssemblyAI**
- 구독자 50명+ 이후, 빠른 응답이 중요해질 때 전환
- 비용: 2분 음성 × 100건/월 = 약 200분 = $1.3~2.2/월 → 무시 가능

## 온보딩 경로 분기 설계

```
음성 분석 결과 → 세그먼트 + 트랙 결정

┌─────────────────────────────────────────┐
│           세그먼트 분기                  │
├──────────────┬──────────────────────────┤
│ 학부모       → Track B 시작             │
│              → 자녀 학년·AI 행동 확인   │
│              → "부모 이중 공포" 공감부터│
├──────────────┬──────────────────────────┤
│ 직장인 일반  → Track A 시작             │
│              → 도구 현황 + 압박 유형    │
├──────────────┬──────────────────────────┤
│ 직장인 대기업│ → Track D 시작           │
│              → 보안 환경 확인 필요      │
│              → "pip 차단 여부" 확인     │
├──────────────┬──────────────────────────┤
│ Excel 집착   → Track C 시작             │
│ 시그널 감지  → 3단계 심리 전환 플로우  │
├──────────────┬──────────────────────────┤
│ 미결정/혼합  → 추가 질문 1~2개 발송    │
│              → 텍스트로 보완 수집       │
└─────────────────────────────────────────┘

긴급도 분기:
- 즉각: "내일 면접/발표/제출" → 당일 원포인트 레슨 제안
- 탐색: 일반 상담 플로우
- 관망: 무료 콘텐츠 먼저 안내 (리드 너처링)
```

## 사용자 경험 (UX) 설계

### 접수 채널

**Phase 1 (간단하게 시작):**
```
카카오채널 또는 구글폼 링크로 음성 파일 업로드
→ "2~5분 동안 편하게 말씀해주세요. 스크립트 없이 그냥 고민을 털어놓으세요."
→ 형식: MP3/M4A/WAV (10MB 이하)
```

**Phase 2 (웹 인터페이스):**
```
Harness 랜딩 페이지에 "녹음 시작" 버튼
→ 브라우저 마이크 → WebRTC → 서버 전송
→ 실시간 또는 비동기 분석
```

### 접수 후 안내 메시지

```
접수 확인 즉시 자동 발송:

"안녕하세요. Harness입니다 🙏
말씀해주신 내용 잘 접수됐습니다.

30분 이내로 분석을 마치고 답변 드리겠습니다.
궁금하신 점이 있으면 이 채널로 바로 말씀해주세요."
```

### 분석 완료 후 첫 답변 구조

```
[공감 → 인식 → 방향 → 다음 단계]

예시:
"말씀하신 내용을 들으니, 정말 답답하셨겠다는 생각이 들었습니다.
'어디까지가 괜찮은지 기준이 없다'는 부분, 많은 부모님들이
똑같이 느끼시는 부분입니다.

연구에 따르면, 이런 상황에서 기준을 세우는 가장 효과적인 방법이
있습니다. 아드님 학년(중2)에 맞춰 구체적으로 말씀드리겠습니다.

먼저 한 가지만 확인드려도 될까요?
학교에서 AI 사용에 대해 어떤 지침을 받으셨나요?
(없다고 하셔도 괜찮습니다 — 그럼 그 상황에 맞는 기준을 드릴게요)"
```

## 구현 계획 (Mac Mini 기반)

### 필요 패키지

```bash
# Mac Mini에 설치 필요
.venv/bin/pip install openai-whisper
.venv/bin/pip install ffmpeg-python  # 오디오 처리

# 시스템 패키지 (Homebrew)
brew install ffmpeg
```

### 스크립트 구조 (Phase 1)

```
scripts/
├── run_voice_onboarding.py     # 음성 파일 → 분석 → 첫 답변 생성
└── voice_onboarding_server.py  # (Phase 2) 실시간 수신 서버
```

`run_voice_onboarding.py`:
```
입력: --audio <파일경로> --customer-id <ID>
처리:
  1. Whisper로 전사
  2. Claude로 세그먼트/트랙/감정 분석
  3. 첫 답변 초안 생성
  4. DB 저장 (raw_signals 또는 별도 onboarding_sessions 테이블)
  5. Slack #edu-onboarding 채널로 대표/부대표에게 통보
출력: onboarding_analysis.json + first_reply_draft.md
```

### DB 스키마 추가 (Phase 1)

```sql
CREATE TABLE onboarding_sessions (
    id SERIAL PRIMARY KEY,
    created_at TIMESTAMP DEFAULT now(),
    customer_id VARCHAR(64),
    audio_path TEXT,                    -- 음성 파일 임시 경로 (삭제 후 NULL)
    transcript TEXT,                    -- 전사 텍스트
    analysis JSONB,                     -- 세그먼트/트랙/감정 등
    first_reply_draft TEXT,             -- 첫 답변 초안
    status VARCHAR(20) DEFAULT 'pending',  -- pending/reviewed/sent
    reviewed_by VARCHAR(20),            -- '대표' or '부대표'
    sent_at TIMESTAMP,
    correlation_id VARCHAR(64) DEFAULT 'edu-consulting-20260524'
);
```
