# EDU 디지털 트윈 코퍼스 명세

> 작성일: 2026-06-13  
> 목적: `evidence-grounded twin`이 선언에 그치지 않도록, 실제 발화 코퍼스의 최소 기준과 추출 규칙을 정의한다.

---

## 1. 기본 원칙

디지털 트윈은 "이럴 것 같다"가 아니라 실제 수집 발화 패턴을 반영해야 한다.

최소 source 범주:

- 맘카페/학부모 커뮤니티
- 블로그
- 유튜브 댓글/설명/클립
- RSS 기사/칼럼
- 직장인 커뮤니티
- 노인 디지털 격차/사기 관련 경험담

---

## 2. 세그먼트별 최소 코퍼스 기준

- 학부모 발화: 500건 이상
- 직장인 발화: 400건 이상
- 성인 자녀 -> 노부모 발화: 200건 이상
- 노인 직접 발화: 150건 이상

초기에는 적더라도, 시뮬레이션 게이트를 본격 운영할 때는 위 기준을 목표로 한다.

---

## 3. 추출할 특징

각 코퍼스에서 최소 아래를 뽑아야 한다.

- 자주 쓰는 어휘
- 질문 시작 방식
- 불안 표현
- 거부감 표현
- 이탈 직전 표현
- "못써먹겠다"류 반응
- 최신 툴/브랜드 언급
- 설치/로그인/실행에서 막히는 서술

---

## 4. twin 생성 방식

트윈 생성 시 최소 아래가 필요하다.

- few-shot 실제 발화 예시
- 세그먼트별 요약 특징
- 금지할 stereotype
- dropout pattern
- selected LLM/tool preference pattern

---

## 5. freshness 규칙

코퍼스는 고정 문서가 아니다.

필수 반영 대상:

- 최신 LLM/앱 버전 변화
- 최근 유튜브 튜토리얼
- 최근 커뮤니티 불만/칭찬 패턴
- 최근 설치/로그인/과금 UX 변화
