## ⭐ 부모 우선 원칙 (2026-05-25 CEO 결정)

> **"교육 대상은 자녀 먼저 보다는 더 관심이 많은 부모(주로 엄마)를 대상으로 하는 교육에 더 중점을 둬. 자녀는 부모가 충분히 AX에 대해 이해가 된 후에 다음 단계."**

### 제품 순서

```
[선수 과정: 필수] 부모 본인 AX 이해 교육 (4주)
       ↓ 완료 후
[본 과정] 자녀 AI 교육 설계 가이드 (4주)
```

**이 순서가 중요한 이유:**
- 부모가 AI를 체험하지 않고는 자녀에게 기준을 줄 수 없음
- 부모의 AX 여정 자체가 감탄 포인트가 됨 ("나도 해보니까 이렇더라")
- 리텐션: 선수 과정 4주 → 본 과정 4주 = 자연스러운 8주 구독

## Track B-0: 부모 본인 AX 이해 과정 (필수 선수 과정)

**대상:** 자녀 교육을 원하는 모든 학부모 — Track B 진입 전 필수

```
Step 1: "AI가 실제로 뭘 잘하고 뭘 못 하는가" 현실 인식
  → 부모에게 가장 많은 오해 두 가지:
     - "AI는 뭐든 다 잘해" (과대 평가)
     - "AI는 위험해, 쓰면 안돼" (과소 평가)
  → 연구 기반 현실: "AI는 도구다. 어떻게 쓰느냐가 결과를 결정한다."

Step 2: 부모 본인이 AI를 직접 써보기
  → 과제: "이번 주 한 가지 일상 업무에 ChatGPT 써보기"
  → 결과 공유: "어떤 점이 좋았나? 어떤 점이 불만족이었나?"
  → 핵심: 직접 경험 없이 자녀에게 AI 기준을 줄 수 없다

Step 3: AI가 인간 사고에 미치는 영향 이해
  → cognitive offloading 이란 무엇인가
  → 어떤 종류의 AI 사용이 사고력을 키우고, 어떤 것이 약화시키는가
  → 연구 인용: "AI 의존은 배움의 속도와 역비례"

Step 4: 우리 가정의 AI 사용 기준 초안
  → "우리 집 AI 사용 헌법" 1페이지 작성
  → 부모가 직접 써야 자녀에게 설득력이 있음
  → 이것이 다음 단계(Track B)의 입력이 됨
```

**감탄 포인트:**
- "내가 써보니까 이런 점이 좋고 이런 점이 위험하더라" — 직접 경험의 권위
- "엄마도 배우고 있어" — 자녀와 함께 성장하는 모습 = 자녀의 신뢰 획득

## 왜 개인화가 Harness의 핵심 무기인가

메가스터디 강의 수강생 50만 명 중 누구도 "이게 내 얘기"라고 느끼지 않는다.
유튜브 AI 교육 채널 구독자 100만 명도 마찬가지다.

**Harness가 독점할 수 있는 포지션:**
> "당신의 상황만을 위한 솔루션"

이것은 규모가 클수록 불가능해진다. Harness가 작을 때만 할 수 있는 것이다.
그리고 이것이 초기 프리미엄 가격을 정당화하는 유일한 근거다.

## 두 가지 맞춤 솔루션 트랙

### 트랙 A: 직장인 업무 자동화 솔루션

**대상:** Tech 비친화 직장인 (30~50대), 팀장·부장급

**플로우:**

```
Step 1: 고민 인테이크 (15분, 비동기 가능)
  → 폼 또는 카카오채널 메시지로 수집
  → 질문:
     Q1. 지금 업무 중 가장 시간이 많이 걸리는 것 3가지?
     Q2. 주로 쓰는 툴은? (Excel / PPT / Notion / 이메일 / 기타)
     Q3. 팀에서 AI 압박을 가장 느끼는 순간은?
     Q4. 가장 두려운 시나리오는? (대체 / 뒤처짐 / 평가 저하)

Step 2: LLM 분석 + 맞춤 솔루션 생성 (자동화 가능)
  → 고민 키워드 분류 → 솔루션 템플릿 매핑 → 직무·툴 맞춤 편집
  → 산출물: "000님 맞춤 AI 도구 가이드" (PDF 또는 노션 페이지)

Step 3: 감탄 포인트 설계
  → 이름 명시: "000 부장님의 경우..."
  → 툴 명시: "엑셀 피벗 테이블 → 이렇게 하면 30분이 3분으로"
  → 직무 명시: "기획팀 주간보고 → ChatGPT 프롬프트 템플릿 동봉"
  → 즉시 실행 가능: "오늘 퇴근 전 5분만 써보세요"
```

**감탄을 만드는 3가지 요소:**

| 요소 | 일반 서비스 | Harness |
|---|---|---|
| 이름 | "여러분" | "김00 부장님" |
| 상황 | "직장인이라면..." | "엑셀 보고서 작업이 많으신 분들..." |
| 행동 | "AI를 써보세요" | "이 프롬프트를 복사해서 엑셀 데이터 붙여넣으세요" |

---

### 트랙 B: 학부모 자녀 사례 맞춤 가이드

**대상:** 초·중등 자녀를 둔 학부모 (30~50대)

**플로우:**

```
Step 1: 사례 의뢰 접수 (구체적 상황 기술)
  → "우리 아이가 이런 상황이에요" 자유 기술
  → 또는 구조화 폼:
     Q1. 자녀 학년?
     Q2. AI를 어떤 방식으로 사용하고 있나요?
     Q3. 가장 걱정되는 상황은?
     Q4. 학교/학원에서 AI 관련 어떤 지시를 받고 있나요?

Step 2: 사례 분류 + 연구 기반 처방 생성
  → 사례 유형 분류 (숙제 의존 / 시험 공부 / 독후감 / 수학 풀이 등)
  → 해당 유형에 적합한 연구 기반 솔루션 매핑
  → "우리 아이 상황" 언어로 재작성

Step 3: 감탄 포인트 설계
  → 아이의 학년과 상황을 정확히 반영한 조언
  → "연구에 따르면" + "그래서 000님 아드님의 경우는"
  → 학부모가 당장 오늘 아이에게 할 수 있는 말 제공
  → "학원에서도 이런 얘기 못 들었을 거예요" 포지셔닝
```

**실제 사례 예시:**

```
의뢰:
"중2 아들이 수학 숙제를 ChatGPT한테 물어보고 답만 베껴요.
 혼내면 '다 아는 건데 왜 또 풀어요'라고 해요."

Harness 맞춤 가이드:
---
000님 아드님 상황을 분석했습니다.

이 패턴을 '정답 바이패스 증후군'이라고 합니다.
답을 알고 있다는 착각이 생기면, 실제 시험에서 막힐 때 더 당황합니다.

연구 근거:
→ Bjork(1994) '바람직한 어려움' 이론: 스스로 틀리는 경험이 장기 기억을 만든다
→ ChatGPT 정답 보기 = 단기 이해 O, 장기 인출 X

지금 당장 이렇게 해보세요:
1. "ChatGPT한테 풀이만 보고, 직접 풀어봐. 30분 줄게." (답 베끼기 금지)
2. 틀렸을 때 → ChatGPT한테 "왜 내 풀이가 틀렸어?" 질문하게 하기
3. 일주일 후 같은 문제 유형 시험처럼 풀게 하기

이 방법의 핵심: AI를 '정답 기계'가 아닌 '오답 해설 도구'로 쓰는 것입니다.
---
```

## 트랙 D: AI 업무 설계법 — 대기업 바이브 코딩 직장인 특화

> 대표님 삼성전자 내부 인텔리전스 (2026-05-25):
> 경영진 "AI로 업무 70% 자동화" 지시 → 전 직원 VS Code + Cline 바이브 코딩 시도
> 3개의 벽: (1) AI 지시력 부족 (2) 설계 사고 부재 (3) 보안 환경 장벽
> 이것이 삼성뿐 아니라 LG·SK·현대·KT 등 전 대기업 공통 현실

### 이 세그먼트의 특수성

코딩 교육이 아니다. **코드를 모르는 사람이 AI에게 제대로 지시하는 법** 교육이다.

WTP 근거: 선택이 아니라 회사 과제(고과 직결) → ₩99,000~₩300,000도 낼 의향

### 벽 1: AI 지시력 부족 → "AI 지시 설계법"

```
❌ 바이브 지시 (현재 대부분):
"이 Excel 파일 자동화해줘"

✅ 설계된 지시 (Harness가 가르치는 것):
"입력: A 시스템 CSV (컬럼: 날짜, 품번, 수량)
 처리: 품번이 B 시스템 목록에 있으면 '승인', 없으면 '검토필요'
 출력: 기존 Excel에 '판정결과' 컬럼 추가, 빨간/초록 색상 표시
 조건: 에러 발생 시 해당 행에 '오류' 기입 후 계속 진행"
→ 같은 AI도 결과물 품질이 완전히 달라진다
```

### 벽 2: 설계 사고 부재 → "Harness Engineering (확장 설계법)"

```
바이브 코딩 전 반드시 그려야 하는 것:
[데이터 입력] → [처리 로직] → [결과 출력] → [다음에 붙일 것]

지금 만드는 기능이 나중에 어디에 연결될지 미리 설계하면
→ 코드 사일로가 아니라 확장 가능한 시스템이 된다
→ 기존 Excel 사일로와 동일한 실수를 코드로 반복하지 않게 된다
```

### 벽 3: 보안 환경 장벽 → "대기업 보안 환경 생존법"

```
pip 차단 → "외부 패키지 없이 Python 기본 라이브러리만으로 구현해줘" 지시법
외부 접근 차단 → "인터넷 연결 없이 로컬에서만 실행되는 코드로" 지시법
Gauss/Cline 한계 → 정밀한 spec 지시로 약한 AI의 결과물 품질 보완법
```

### Pretotyping 카피 (대기업 직원 전용)

> "회사에서 AI 자동화 과제 하라는데, 막막하신가요?
>  VS Code 켜놓고 Cline한테 말했더니 엉뚱한 게 나오거나,
>  pip install이 안 되거나, 만들었는데 다음 주에 고칠 수가 없거나.
>  코딩 몰라도 됩니다. AI에게 제대로 지시하는 법을 알면 됩니다."

## 트랙 C: Excel 신주단지 증후군 해방 컨설팅

> 대표님 통찰 (2026-05-25):
> "직장인들이 자신만의 파편화된 Excel을 신주단지 모시듯 관리하지만
>  결국 폐기될 수밖에 없는 포맷이다. 이 덫에서 빠져나오게 해야 한다."

### 왜 Excel을 못 버리는가 — 심리적 분석

표면적 이유: "내 업무 방식이 다 여기 있어"
실제 이유: **정보 독점을 통한 고용 안정 전략** (의식적이든 무의식적이든)

| 심리 메커니즘 | 내용 |
|---|---|
| 영역 표시 (Territoriality) | "이 파일은 내 것" = 직업적 영역 |
| 불가결성 확보 | "나만 이걸 알아" = 해고 방어 |
| 전환 비용 과대평가 | "바꾸면 다 날아간다" (실제로는 안 그럼) |
| 현상 유지 편향 | 불편해도 익숙한 게 낫다 |

### 절대 하면 안 되는 접근

> "Excel 버리고 Notion 쓰세요."

→ 즉각적 거부 반응. 정체성·안전감 공격으로 받아들임. 상담 즉시 종료.

### 올바른 접근 — 3단계 심리적 전환

```
1단계: 인정 (저항 없애기)
   "Excel 잘 쓰시는 거 맞아요. 그 구조가 다 이유가 있는 겁니다."
   → 신주단지를 부정하지 않는다.

2단계: 확장 (Excel을 더 강하게)
   "이 Excel에 AI를 연결하면 수작업이 사라집니다."
   → Excel을 버리는 게 아니라 업그레이드하는 것으로 포지셔닝.

3단계: 자연스러운 이탈 (결과로 증명)
   자동화 결과를 직접 경험하면 → 자발적으로 더 나은 도구로 이동.
```

### 회사 규모별 도구 제약 — 반드시 먼저 확인

## 1. 왜 사주/타로인가 — Harness와의 심리적 동형성

| 사주/타로 방문 고객 심리 | Harness 잠재 고객 심리 |
|---|---|
| "뭔가 잘못되고 있는 것 같아" | "AI 시대에 뭔가 놓치고 있는 것 같아" |
| "어떻게 해야 할지 모르겠어" | "AI 써야 하는지, 어떻게 써야 하는지 모르겠어" |
| "혼자 판단하기 두려워" | "내 아이에게 맞는 기준을 모르겠어" |

## 2. 역술인의 상담 7단계 — 심리 메커니즘 분석

### 2-1. 포착 (Observation) — 말하기 전에 이미 정보 수집

```
실제 역술인:
고객이 들어오는 순간 → 옷차림·표정·걸음걸이·손 상태·눈빛 관찰
→ "이 분은 최근 스트레스가 많은 분이다" 내부 판단 완료

Harness 온보딩 적용:
사용자가 첫 화면에서 클릭하는 패턴 관찰
→ "AI가 걱정돼요" vs "AI를 배우고 싶어요" 중 무엇을 먼저 클릭하는가
→ 클릭 순서, 망설임, 이탈 후 재진입 등 행동 데이터가 이미 프로파일
```

### 2-2. 진입 허들 최소화 (Ease of Entry)

```
실제 역술인:
"언제 태어나셨어요?" — 단 하나의 질문으로 시작
부담 없는 첫 질문 → 고객의 경계심 해제

한국 앱 패턴:
점신: 가벼운 무료 운세 → 자연스럽게 상담사 연결
포스텔러: 무료 콘텐츠 먼저 → 원하면 유료 확장
→ 공통: "일단 한 번 해봐" 진입 장벽 제거

Harness 적용:
첫 화면 질문 단 3개 (90초 이내 완성)
"결제 없이 진단 먼저 받아보세요"
```

### 2-3. 고확률 진술 (Barnum/Forer Effect)

이것이 가장 핵심적인 기법이다.

```
심리학적 정의:
Barnum/Forer Effect — 모호하지만 개인적으로 느껴지는 진술에
사람들은 "이건 나에 대한 얘기"라고 강하게 반응한다.

역술인 실제 발화:
"최근에 중요한 결정을 앞두고 계신 것 같아요."
→ 거의 모든 사람에게 해당하지만, 듣는 사람은 "어떻게 알았지?" 반응

Harness 학부모 버전 고확률 진술 예시:
"중학교 자녀를 두신 분들이 대부분 이런 고민을 하세요.
 'AI를 쓰면 사고력이 떨어질까봐', 동시에 '안 쓰면 뒤처질까봐'.
 두 걱정이 동시에 드신 적 있으시죠?"
→ 90%의 학부모에게 정확히 해당. 하지만 듣는 사람은 "나만 이랬나?" 느낌.

Harness 직장인 버전 고확률 진술 예시:
"팀에서 누군가가 AI로 뭔가를 빠르게 해냈을 때,
 '나만 뒤처지는 건 아닌가' 하는 느낌 받으신 적 있으세요?"
→ 거의 모든 직장인에게 해당.
```

**Barnum Effect 작동 조건 (Harness가 반드시 지켜야 할 것):**
1. 긍정·부정이 섞여야 함 ("잘 하고 계시지만, 이 부분에서...")
2. 구체적인 숫자나 상황이 포함돼야 함 ("중학교 1학년" 같은 맥락)
3. "대부분이" → "특히 000 같은 상황에서는" 순서로 이동

### 2-4. 반응 기반 조정 (Client Feedback Loop)

```
역술인 기법:
진술 → 고객 반응 관찰 → 맞으면 강화 / 틀리면 빠르게 전환

실제 대화:
역술인: "요즘 집안에서 갈등이 좀 있는 것 같은데요?"
고객: "음... 사실 직장 쪽이 더 문제예요."
역술인: "아, 사주상으로 직장 궁이 특히 요동치고 있어요.
         그게 가족에도 영향을 주는 거예요."
→ 틀린 것을 빠르게 회수하고 맞는 방향으로 통합

Harness 디지털 버전:
선택지 기반 분기 질문
"다음 중 더 와닿는 상황은?"
  A. "아이가 ChatGPT로 숙제를 그냥 베껴요"
  B. "아이가 AI 없이는 아무것도 못 하는 것 같아요"
  C. "학교에서는 쓰지 말라는데, 집에서는 어떻게 해야 할지"
  D. "다른 아이들은 AI로 뭔가를 한다는데 우리 애만 뒤처지는 것 같아요"
→ 고객이 선택 = 스스로 pain point 특정
→ 이후 처방이 자동으로 개인화
```

### 2-5. 고객이 정보를 채우게 만들기 (Client Participation)

```
역술인 핵심 기법:
"최근에 누군가와 관계가 어떠셨나요?" (모호한 질문)
→ 고객이 구체적 상황을 말함
→ 역술인은 그것을 가져다가 "사주에도 그게 나와 있어요"라고 돌려줌

심리학: 사람들은 자신이 말한 것을 남이 알아챈 것처럼 느낀다

Harness 적용:
"어떤 상황이 가장 걱정되세요? 자유롭게 말씀해주세요."
→ 고객이 말한 것을 Claude가 분류 후 처방에 그대로 반영
→ 처방에 "말씀하신 것처럼 [자녀가 ChatGPT로 답 베끼는 상황]에서..."
→ 고객: "어떻게 내 상황을 이렇게 정확히 알지?"
```

### 2-6. 처방의 부분 공개 (Teaser → Full Reveal)

```
역술인 기법:
"운세상 이 방향이 맞긴 한데..."
"이 부분은 조금 더 자세히 봐야 해요"
→ 다음 방문, 추가 상담 유도

실제 앱 패턴:
Kasamba: 3분 무료 → 이후 유료
Keen: 5분 $1 → 이후 정가
포스텔러: 무료 콘텐츠 → 유료 상세 리포트
마이파이: 기질 분석 무료 → 상세 직업 적합도 유료

Harness 적용:
무료 진단 결과: "[아이 유형] + 처방 2가지 (무료)"
유료 전환 포인트: "이 경우 더 중요한 3번째 요소는..."
→ 첫 결제: 원포인트 레슨 ₩9,900 (정가 ₩49,000)
→ 또는: "4주 부모 AX 과정 무료 체험 2주"
```

### 2-7. 레이블링과 소속감 (Labeling & Identity)

```
역술인 기법:
"000님은 ○○띠라서 이런 특성이 있어요"
"이 사주에는 특별한 기운이 있어요"
→ 사람들은 레이블을 받으면 그 레이블에 맞게 행동하고,
   그 레이블을 내 것으로 받아들임

성공 사례:
Co-Star: 별자리 기반 정밀 성격 분석 → "나 INFJ야"처럼 공유
MBTI: 레이블 자체가 소셜 콘텐츠가 됨

Harness 레이블링 시스템:
[부모 유형]
- "AI 이중 공포형" — AI 쓰면 걱정, 안 쓰면 더 걱정
- "AI 관망형 부모" — 지켜보는 중이지만 기준이 없음
- "AI 주도형 부모" — 먼저 배우려 하지만 어디서 시작해야 할지 모름

[아이 유형]
- "정답 바이패스 유형" — ChatGPT로 답만 복사
- "AI 의존 발아 단계" — 아직 초기지만 패턴 고착 전 골든 타임
- "AI 협업 성숙 단계" — 올바른 사용 중, 고도화 필요

→ 이 레이블이 바이럴 콘텐츠가 됨 ("우리 아이 AI 의존 유형 알아보기")
→ SNS 공유 → 유입 루프
```

## 4. Harness 온보딩 설계 — "사주 모델" 완전판

### Step 0: 포착 (Observation)
고객이 랜딩 페이지에서 보이는 행동 데이터 수집
- 어떤 카피에서 멈추는가 (학부모 vs 직장인)
- 어떤 버튼을 먼저 클릭하는가 (AI 불안 vs AI 배우기)
- 이탈 후 재방문 패턴

### Step 1: 비침습적 3문항 (90초)

```
Q1. [세그먼트 분류 — 역술인의 "생년월일"]
    저는...
    ○ 초·중·고 자녀를 둔 부모
    ○ AI 압박을 받는 직장인
    ○ AI 시대 커리어가 걱정되는 직장인
    ○ 부모님의 AI 소외가 걱정되는 자녀

Q2. [Pain Point 분류 — 역술인의 "오늘 무슨 일로 오셨어요"]
    지금 가장 걱정되는 것은?
    ○ 아이가 AI에 너무 의존하는 것 같아요
    ○ 아이가 AI를 너무 안 써서 뒤처질까봐
    ○ 무엇이 옳은지 기준이 전혀 없어요
    ○ 나 자신이 AI를 모르니까 아이를 가이드 못 해요 ← (부모 먼저 원칙)

Q3. [온도 파악 — 역술인의 "얼마나 급하신가요"]
    AI에 대해 직접 써보신 경험이 있으신가요?
    ○ 거의 없어요 (이제 막 시작)
    ○ 가끔 써봤어요 (아직 익숙하지 않음)
    ○ 꽤 써봤어요 (쓰는데 뭔가 부족한 느낌)
```

### Step 2: "맞아요 순간" 연출 (Barnum Effect)

**입력값 → 고확률 진술 자동 생성 (LLM)**

```
예시 — Q1: 학부모 / Q2: 기준이 없음 / Q3: 가끔 써봤음

출력:
"[아, 저도 잘 알 것 같은 상황이에요]

자녀가 AI를 쓰는 걸 봤을 때,
혼내야 할지 내버려둬야 할지 판단이 안 서신 적 있으시죠?

주변 엄마들은 "AI 시켜야 한다", "AI가 망친다"로 나뉘는데
정작 우리 아이에게 뭐가 맞는지는 아무도 안 알려주는 것 같고.

그 답답함, 완전히 이해합니다.
사실 그게 당연한 이유가 있어요."

→ 고객이 느끼는 것: "어떻게 내 상황을 이렇게 정확히 알지?"
→ 실제: 이 상황은 90%의 학부모에게 해당하는 고확률 진술
```

### Step 3: 고객이 정보를 채우게 만들기

```
"조금 더 여쭤봐도 될까요?"

"최근에 아이가 AI를 사용한 구체적인 상황이 있으셨나요?
 (예: 숙제에 사용, 질문에 사용, 창작에 사용 등)"
                   ↓ 자유 기술 or 선택지
"어떤 반응이 가장 걱정되셨나요?"
 ○ 아이가 AI 없이는 시작을 못 해요
 ○ AI 답변을 그대로 제출해요
 ○ 아이가 AI를 어떻게 쓰는지 제가 모르겠어요
 ○ 선생님이 AI 쓰지 말라고 했는데 몰래 써요
```

### Step 4: 무료 처방 일부 공개 (Teaser)

```
"000님 상황 분석 완료"

━━━━━━━━━━━━━━━━━━━━━━
[진단 결과]
아이 유형: AI 의존 발아 단계 ⚠️
부모 유형: AI 기준 부재형 📋
━━━━━━━━━━━━━━━━━━━━━━

지금 상황에서 가장 중요한 것 3가지:

✅ 1. [오늘 당장 할 수 있는 것]
   "AI를 왜 쓰는지 아이에게 한 번 물어보세요."
   이 질문만으로도 아이의 인식이 달라집니다.

✅ 2. [이번 주 안에 해야 할 것]
   "이 유형의 아이에게는 이렇게 접근하면..."
   ← 이 부분은 상황별로 다르게 설계해야 해서
      맞춤 가이드에서 자세히 설명드릴게요

🔒 3. [가장 중요한 핵심 — 이게 안 되면 1, 2도 무효]
   "000님 아이에게 특히 맞는 접근법은..."
   ← 이 부분은 000님 아이의 구체적 상황을 더 알아야 해요
━━━━━━━━━━━━━━━━━━━━━━

[지금 바로 받아보실 수 있어요]
→ 맞춤 상담 첫 번째: ₩9,900 (정가 ₩49,000)
   "오늘 받아보신 진단을 바탕으로 000님 아이만을 위한 처방"
또는
→ 4주 부모 AX 과정 2주 무료 체험
   "부모님이 먼저 AI를 이해한 후 아이를 가이드하는 법"
```

### Step 5: 레이블 + 공유 유도 (바이럴 설계)

```
카드 형태로 공유 가능:

┌─────────────────────────┐
│  📋 나의 AI 부모 유형   │
│  ──────────────────── │
│  AI 기준 부재형         │
│  "알고 싶지만           │
│   어디서 시작할지       │
│   모르는 부모"         │
│                          │
│  우리 아이 유형:         │
│  AI 의존 발아 단계 ⚠️   │
│  → 지금이 골든 타임     │
│  ──────────────────── │
│  by Harness            │
└─────────────────────────┘

[내 결과 공유하기] → 카카오톡·인스타 → 바이럴
[자세한 처방 받기] → 유료 전환
```

## 5. 핵심 카피 원칙 (역술인 언어를 Harness에 이식)

| 역술인 언어 | Harness 번역 |
|---|---|
| "사주상으로 이 시기가..." | "연구에 따르면 이 나이 단계에서는..." |
| "당신의 기운이..." | "000님 아이의 학습 패턴이..." |
| "이건 당신만의 문제가 아니에요" | "이런 고민을 하는 부모님이 전국에 수백만 명이에요" |
| "지금 이 시기가 중요해요" | "이 시기는 AI 의존 패턴이 굳어지기 전 골든 타임이에요" |
| "다음에 한 번 더 오세요" | "3개월 후 어떻게 달라졌는지 점검해드릴게요" |
| "이 방향이 맞긴 한데..." | "처방 방향은 맞아요. 다만 000님 아이의 경우 특히..." |

## 6. Harness가 훼손하면 안 되는 본질

사주/타로를 벤치마킹하되, 아래는 절대 훼손하지 않는다:

| 사주/타로 방식 (하지 않을 것) | Harness 본질 (지킬 것) |
|---|---|
| 근거 없는 운명론 ("당신의 운이...") | 연구 기반 증거 인용 |
| 공포 조장 ("이걸 안 하면 큰일 나요") | 손실 언어는 쓰되, 과장 없이 |
| 결과의 불확실성 덮기 | "이 방법이 100% 효과 있다"는 보장 금지 |
| 무한 방문 유도 (의존 조장) | 자립 목표: "이것을 알면 혼자 판단 가능" |
| 모호한 일반 처방 | 구체적·실행 가능한 처방 |
| 소수에게만 유효한 클로즈드 정보 | 연구는 공개된 것, Harness는 해석을 파는 것 |

**핵심 차이:**
사주/타로는 "나만 아는 신비한 정보"를 파는 척한다.
Harness는 "공개된 연구를 당신 상황에 맞게 번역"하는 것을 판다.

방식은 사주를 따르되, 내용은 과학을 따른다.

## 7. A/B 테스트 우선순위

1. **Step 2 Barnum 진술 버전 비교:**
   - A: "많은 부모님들이..."
   - B: "중학교 1학년 자녀를 둔 부모님들이..."
   → 구체적 맥락이 Barnum Effect를 강화하는지 검증

2. **Step 4 Teaser 강도 비교:**
   - A: 처방 1개 무료 + 2개 잠금
   - B: 처방 0개 + 진단만 무료
   → 처방 일부 공개가 전환율에 미치는 영향

3. **Step 5 레이블 공유율 비교:**
   - A: 부모 유형 레이블
   - B: 아이 유형 레이블
   → 어떤 레이블이 더 많이 공유되는가

4. **첫 결제 구조 비교:**
   - A: 원포인트 레슨 ₩9,900 (Keen 모델)
   - B: 2주 무료 후 구독 ₩29,900 (Free Trial 모델)
   → LTV 기준으로 비교

## 8. 옵션 기능 — 사주 기질 분석 통합 (CEO 결정 2026-05-25)

> "희망자에 한해 내부 사주 분석 로직을 돌려 이 사람의 성향을 반영하여 컨설팅에 활용한다."
> — 대표(CEO), 2026-05-25

### 전략적 근거

- 점신 1,800만 이용자가 증명: 한국 30~50대 학부모는 중요한 결정 앞에서 사주를 실제 참고
- 교육 결정이 가장 중요한 결정 중 하나 → 사주 참고 욕구가 가장 높은 영역
- AI 교육 컨설팅 + 사주 기질 분석 조합 = 현재 시장에 없음

### 포지셔닝 원칙 (본질 훼손 방지)

```
❌ 하지 않는 것:
"사주상 AI 교육이 맞습니다" (운명론)
"이 띠는 AI를 잘 씁니다" (허위 과학)

✅ 하는 것:
"많은 분들이 참고하는 오행 기질 프레임으로 아이의 학습 성향을 파악합니다"
"MBTI처럼, 과학적으로 완전히 검증된 건 아니지만 기질 참고자료로 활용합니다"
연구 기반 처방이 본체, 사주 기질이 맥락 보강
```

### 사주 → 학습 기질 매핑 로직

**오행(五行) 기반 학습 성향:**

| 오행 | 천간·지지 | 기질 특성 | 학습 스타일 | AI 교육 시사점 |
|---|---|---|---|---|
| **木(목)** | 甲乙·寅卯 | 창의적, 성장 지향, 자율적 | 자유 탐구형. 틀에 박힌 교육 거부 | AI를 창의 도구로 → 의존 위험 높음. 독립 탐구 설계 필수 |
| **火(화)** | 丙丁·巳午 | 열정적, 표현력 강함, 사교적 | 발표·토론·프레젠테이션 선호 | AI 발표자료 과의존 주의. 구두 표현력 별도 강화 |
| **土(토)** | 戊己·辰戌丑未 | 안정적, 신중, 계획적 | 반복 학습, 체계적 커리큘럼 선호 | AI 루틴화 위험. 다양한 접근법 의도적 노출 필요 |
| **金(금)** | 庚辛·申酉 | 논리적, 분석적, 원칙 중시 | 수학·과학·논리 영역 강함 | AI 비판적 검증에 적합. 결과보다 과정 질문 훈련 |
| **水(수)** | 壬癸·亥子 | 지혜롭고 유연, 깊은 사고 | 독서·탐구, 깊이 있는 학습 | AI 의존 대신 독서·자기 사유 병행 설계 |

**사주 계산 방식:**
```
입력: 생년월일시 (음력/양력 모두 지원)
처리:
  1. 만세력(萬歲曆) 기반 사주팔자(四柱八字) 산출
  2. 년주·월주·일주·시주 각각의 오행 추출
  3. 오행 비율 산정 (목:화:토:금:수 %)
  4. 주도 오행 1~2개 기반 기질 프로파일 생성

출력 예시:
  "목(木) 65% / 화(火) 20% / 기타 15%
   → 창의·자율 기질이 강한 유형
   → AI 교육에서 특히 주의할 점: ..."
```

**오픈소스 만세력 라이브러리:**
- Python: `korean-lunar-calendar`, `saju` 패키지 (PyPI)
- 직접 구현: 1900~2100년 만세력 테이블 + 오행 매핑 딕셔너리

## 왜 음성 온보딩인가

**텍스트 폼의 한계:**
```
폼 질문: "AI를 어떤 방식으로 사용하고 있나요?"
→ 대부분 고객: 한두 줄 답변 → 맥락 부족 → 맞춤화 부정확

실제 고민은 이렇게 나온다:
"음... 우리 아이가요, 중2인데, 요즘 수학 숙제를 계속
 ChatGPT한테 물어봐요. 근데 저도 잘 모르거든요,
 어디까지가 괜찮고 어디부터가 문제인지. 학교에서
 뭐라고 하는지도 모르고... 그냥 불안한 거죠."
```

이 발화에 들어있는 정보:
- 세그먼트: 학부모 (Track B)
- 자녀: 중2
- 우려: ChatGPT 숙제 의존 + 경계 기준 불명확
- 감정 상태: 불안, 정보 부재
- 온보딩 진입점: "AI 의존 기준선 설정" + "학교 규칙 안내"

이 정보를 텍스트 폼으로 얻으려면 10개 질문이 필요하다. 음성으로 2분이면 충분하다.

   → 핵심 고민 3가지 추출
   → 감정 상태 태깅 (공황/호기심/체념/분노/혼란)
   → 긴급도 판단 (즉각 처방 필요 / 탐색 단계)

## STT 엔진 비교

| 항목 | Whisper (Local) | AssemblyAI | Deepgram |
|---|---|---|---|
| **비용** | 무료 (GPU 필요) | $0.37~$0.65/시간 | $0.0043/분 |
| **한국어 지원** | ✅ 우수 | ✅ 양호 | ⚠️ 제한적 |
| **속도** | 느림 (CPU 기준 2~5배 실시간) | 빠름 (실시간의 1/5) | 빠름 |
| **프라이버시** | ✅ 로컬 처리 (데이터 외부 전송 없음) | ⚠️ 클라우드 | ⚠️ 클라우드 |
| **정확도 (한국어)** | ★★★★☆ | ★★★★☆ | ★★★☆☆ |
| **세팅 복잡도** | 높음 | 낮음 (API) | 낮음 (API) |

**Phase 1 권고: Whisper local**
- 이유 1: 고객 음성 데이터는 개인 고민이 담겨 있어 프라이버시 민감
- 이유 2: 비용 무료 — Phase 1 규모에서 API 비용 절약
- 이유 3: Mac Mini M4에 설치 가능 (`pip install openai-whisper`)
- 단점: CPU만으로는 2분 음성 변환에 3~8분 소요 → 실시간 불가. "접수 후 30분 내 분석" 약속으로 해결.

**Phase 2 업그레이드: AssemblyAI**
- 구독자 50명+ 이후, 빠른 응답이 중요해질 때 전환
- 비용: 2분 음성 × 100건/월 = 약 200분 = $1.3~2.2/월 → 무시 가능

## 후보 카피 A — "불안 해소형" (현재 위기 프레이밍)

### 헤드라인
> **"AI가 우리 아이 공부를 대신 하고 있다면?"**

### 서브헤드
> 클로바노트로 수업 녹음, ChatGPT로 과제 처리 — 지금 이 습관이 10년 후를 결정합니다.
> 매달 AI 교육 연구를 분석해 **우리 아이 상황에 맞는 대응 전략**을 드립니다.

### 가치 제안 (3줄)
- AI 의존 없이 생각하는 힘을 키우는 학습 설계
- 최신 AI 교육 연구 기반, 학부모가 바로 적용할 수 있는 전략
- 월 1회 맞춤 진단 + 상시 Q&A

---

## 후보 카피 B — "성과형" (경쟁력 언어)

### 헤드라인
> **"AI 시대, 우리 아이만의 경쟁력은 뭔가요?"**

### 서브헤드
> AI를 쓰는 아이들이 넘쳐나는 시대.
> 결국 살아남는 아이는 **AI 없이도 생각할 수 있는 아이**입니다.
> 매달 연구 기반 학습 전략을 제공하는 상시 자문 구독 서비스입니다.

### 가치 제안 (3줄)
- 전 세계 AI 교육 연구를 우리 아이 학교생활에 적용
- 학원 스케줄 아닌, **학습 전략** 설계에 집중
- 부모가 알아야 할 AI 교육 동향, 매달 30분 안에 소화

## 후보 카피 C — "진단형" (Pretotyping 최적화)

### 헤드라인
> **"우리 아이 AI 의존도, 한번 진단해 볼까요?"**

### 서브헤드
> 5분 체크리스트로 지금 학습 패턴을 파악하고,
> **AI 시대에 맞는 다음 학습 전략**을 제안해드립니다.

### CTA (베타 참여 버튼)
> "무료 진단 신청하기 (베타 참여 · 선착순 10명)"

### 가치 제안 (진단 후 유료 전환)
- 진단 결과: AI 의존도 점수 + 권장 학습 조정 3가지
- 구독 시: 매달 업데이트 진단 + 상시 자문

## A/B 테스트 설계

| 변수 | 테스트 A | 테스트 B |
|---|---|---|
| 프레임 | 불안 해소 (카피 A) | 성과·경쟁력 (카피 B) |
| CTA | "AI 의존 없는 학습 신청" | "자녀 경쟁력 전략 받기" |
| 측정 지표 | 클릭률 (CTR) | 이메일 등록률 |
| 기간 | 각 1주 (순차) | 각 1주 (순차) |
| 예산 | 인스타 광고 $15 × 2회 | $30 총 |
| 판단 기준 | CTR > 2% → 해당 카피 선택 | CTR < 2% → 카피 C(진단형) 시도 |
