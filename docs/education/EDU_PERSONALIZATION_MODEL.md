## ⭐ 부모 우선 원칙 (2026-05-25 CEO 결정)

> **"교육 대상은 자녀 먼저 보다는 더 관심이 많은 부모(주로 엄마)를 대상으로 하는 교육에 더 중점을 둬. 자녀는 부모가 충분히 AX에 대해 이해가 된 후에 다음 단계."**

### 제품 순서

```
[선수 과정: 필수] 부모 본인 AX 이해 교육 (4주)
       ↓ 완료 후
[본 과정] 자녀 AI 교육 설계 가이드 (4주)
```

**이 순서가 중요한 이유:**
- 부모가 AI를 체험하지 않고는 자녀에게 기준을 줄 수 없음
- 부모의 AX 여정 자체가 감탄 포인트가 됨 ("나도 해보니까 이렇더라")
- 리텐션: 선수 과정 4주 → 본 과정 4주 = 자연스러운 8주 구독

## Track B-0: 부모 본인 AX 이해 과정 (필수 선수 과정)

**대상:** 자녀 교육을 원하는 모든 학부모 — Track B 진입 전 필수

```
Step 1: "AI가 실제로 뭘 잘하고 뭘 못 하는가" 현실 인식
  → 부모에게 가장 많은 오해 두 가지:
     - "AI는 뭐든 다 잘해" (과대 평가)
     - "AI는 위험해, 쓰면 안돼" (과소 평가)
  → 연구 기반 현실: "AI는 도구다. 어떻게 쓰느냐가 결과를 결정한다."

Step 2: 부모 본인이 AI를 직접 써보기
  → 과제: "이번 주 한 가지 일상 업무에 ChatGPT 써보기"
  → 결과 공유: "어떤 점이 좋았나? 어떤 점이 불만족이었나?"
  → 핵심: 직접 경험 없이 자녀에게 AI 기준을 줄 수 없다

Step 3: AI가 인간 사고에 미치는 영향 이해
  → cognitive offloading 이란 무엇인가
  → 어떤 종류의 AI 사용이 사고력을 키우고, 어떤 것이 약화시키는가
  → 연구 인용: "AI 의존은 배움의 속도와 역비례"

Step 4: 우리 가정의 AI 사용 기준 초안
  → "우리 집 AI 사용 헌법" 1페이지 작성
  → 부모가 직접 써야 자녀에게 설득력이 있음
  → 이것이 다음 단계(Track B)의 입력이 됨
```

**감탄 포인트:**
- "내가 써보니까 이런 점이 좋고 이런 점이 위험하더라" — 직접 경험의 권위
- "엄마도 배우고 있어" — 자녀와 함께 성장하는 모습 = 자녀의 신뢰 획득

## 왜 개인화가 Harness의 핵심 무기인가

메가스터디 강의 수강생 50만 명 중 누구도 "이게 내 얘기"라고 느끼지 않는다.
유튜브 AI 교육 채널 구독자 100만 명도 마찬가지다.

**Harness가 독점할 수 있는 포지션:**
> "당신의 상황만을 위한 솔루션"

이것은 규모가 클수록 불가능해진다. Harness가 작을 때만 할 수 있는 것이다.
그리고 이것이 초기 프리미엄 가격을 정당화하는 유일한 근거다.

## 두 가지 맞춤 솔루션 트랙

### 트랙 A: 직장인 업무 자동화 솔루션

**대상:** Tech 비친화 직장인 (30~50대), 팀장·부장급

**플로우:**

```
Step 1: 고민 인테이크 (15분, 비동기 가능)
  → 폼 또는 카카오채널 메시지로 수집
  → 질문:
     Q1. 지금 업무 중 가장 시간이 많이 걸리는 것 3가지?
     Q2. 주로 쓰는 툴은? (Excel / PPT / Notion / 이메일 / 기타)
     Q3. 팀에서 AI 압박을 가장 느끼는 순간은?
     Q4. 가장 두려운 시나리오는? (대체 / 뒤처짐 / 평가 저하)

Step 2: LLM 분석 + 맞춤 솔루션 생성 (자동화 가능)
  → 고민 키워드 분류 → 솔루션 템플릿 매핑 → 직무·툴 맞춤 편집
  → 산출물: "000님 맞춤 AI 도구 가이드" (PDF 또는 노션 페이지)

Step 3: 감탄 포인트 설계
  → 이름 명시: "000 부장님의 경우..."
  → 툴 명시: "엑셀 피벗 테이블 → 이렇게 하면 30분이 3분으로"
  → 직무 명시: "기획팀 주간보고 → ChatGPT 프롬프트 템플릿 동봉"
  → 즉시 실행 가능: "오늘 퇴근 전 5분만 써보세요"
```

**감탄을 만드는 3가지 요소:**

| 요소 | 일반 서비스 | Harness |
|---|---|---|
| 이름 | "여러분" | "김00 부장님" |
| 상황 | "직장인이라면..." | "엑셀 보고서 작업이 많으신 분들..." |
| 행동 | "AI를 써보세요" | "이 프롬프트를 복사해서 엑셀 데이터 붙여넣으세요" |

---

### 트랙 B: 학부모 자녀 사례 맞춤 처방

**대상:** 초·중등 자녀를 둔 학부모 (30~50대)

**플로우:**

```
Step 1: 사례 의뢰 접수 (구체적 상황 기술)
  → "우리 아이가 이런 상황이에요" 자유 기술
  → 또는 구조화 폼:
     Q1. 자녀 학년?
     Q2. AI를 어떤 방식으로 사용하고 있나요?
     Q3. 가장 걱정되는 상황은?
     Q4. 학교/학원에서 AI 관련 어떤 지시를 받고 있나요?

Step 2: 사례 분류 + 연구 기반 처방 생성
  → 사례 유형 분류 (숙제 의존 / 시험 공부 / 독후감 / 수학 풀이 등)
  → 해당 유형에 적합한 연구 기반 솔루션 매핑
  → "우리 아이 상황" 언어로 재작성

Step 3: 감탄 포인트 설계
  → 아이의 학년과 상황을 정확히 반영한 조언
  → "연구에 따르면" + "그래서 000님 아드님의 경우는"
  → 학부모가 당장 오늘 아이에게 할 수 있는 말 제공
  → "학원에서도 이런 얘기 못 들었을 거예요" 포지셔닝
```

**실제 사례 예시:**

```
의뢰:
"중2 아들이 수학 숙제를 ChatGPT한테 물어보고 답만 베껴요.
 혼내면 '다 아는 건데 왜 또 풀어요'라고 해요."

Harness 맞춤 처방:
---
000님 아드님 상황을 분석했습니다.

이 패턴을 '정답 바이패스 증후군'이라고 합니다.
답을 알고 있다는 착각이 생기면, 실제 시험에서 막힐 때 더 당황합니다.

연구 근거:
→ Bjork(1994) '바람직한 어려움' 이론: 스스로 틀리는 경험이 장기 기억을 만든다
→ ChatGPT 정답 보기 = 단기 이해 O, 장기 인출 X

지금 당장 이렇게 해보세요:
1. "ChatGPT한테 풀이만 보고, 직접 풀어봐. 30분 줄게." (답 베끼기 금지)
2. 틀렸을 때 → ChatGPT한테 "왜 내 풀이가 틀렸어?" 질문하게 하기
3. 일주일 후 같은 문제 유형 시험처럼 풀게 하기

이 방법의 핵심: AI를 '정답 기계'가 아닌 '오답 해설 도구'로 쓰는 것입니다.
---
```

## 트랙 D: AI 업무 설계법 — 대기업 바이브 코딩 직장인 특화

> 대표님 삼성전자 내부 인텔리전스 (2026-05-25):
> 경영진 "AI로 업무 70% 자동화" 지시 → 전 직원 VS Code + Cline 바이브 코딩 시도
> 3개의 벽: (1) AI 지시력 부족 (2) 설계 사고 부재 (3) 보안 환경 장벽
> 이것이 삼성뿐 아니라 LG·SK·현대·KT 등 전 대기업 공통 현실

### 이 세그먼트의 특수성

코딩 교육이 아니다. **코드를 모르는 사람이 AI에게 제대로 지시하는 법** 교육이다.

WTP 근거: 선택이 아니라 회사 과제(고과 직결) → ₩99,000~₩300,000도 낼 의향

### 벽 1: AI 지시력 부족 → "AI 지시 설계법"

```
❌ 바이브 지시 (현재 대부분):
"이 Excel 파일 자동화해줘"

✅ 설계된 지시 (Harness가 가르치는 것):
"입력: A 시스템 CSV (컬럼: 날짜, 품번, 수량)
 처리: 품번이 B 시스템 목록에 있으면 '승인', 없으면 '검토필요'
 출력: 기존 Excel에 '판정결과' 컬럼 추가, 빨간/초록 색상 표시
 조건: 에러 발생 시 해당 행에 '오류' 기입 후 계속 진행"
→ 같은 AI도 결과물 품질이 완전히 달라진다
```

### 벽 2: 설계 사고 부재 → "Harness Engineering (확장 설계법)"

```
바이브 코딩 전 반드시 그려야 하는 것:
[데이터 입력] → [처리 로직] → [결과 출력] → [다음에 붙일 것]

지금 만드는 기능이 나중에 어디에 연결될지 미리 설계하면
→ 코드 사일로가 아니라 확장 가능한 시스템이 된다
→ 기존 Excel 사일로와 동일한 실수를 코드로 반복하지 않게 된다
```

### 벽 3: 보안 환경 장벽 → "대기업 보안 환경 생존법"

```
pip 차단 → "외부 패키지 없이 Python 기본 라이브러리만으로 구현해줘" 지시법
외부 접근 차단 → "인터넷 연결 없이 로컬에서만 실행되는 코드로" 지시법
Gauss/Cline 한계 → 정밀한 spec 지시로 약한 AI의 결과물 품질 보완법
```

### Pretotyping 카피 (대기업 직원 전용)

> "회사에서 AI 자동화 과제 하라는데, 막막하신가요?
>  VS Code 켜놓고 Cline한테 말했더니 엉뚱한 게 나오거나,
>  pip install이 안 되거나, 만들었는데 다음 주에 고칠 수가 없거나.
>  코딩 몰라도 됩니다. AI에게 제대로 지시하는 법을 알면 됩니다."

## 트랙 C: Excel 신주단지 증후군 해방 컨설팅

> 대표님 통찰 (2026-05-25):
> "직장인들이 자신만의 파편화된 Excel을 신주단지 모시듯 관리하지만
>  결국 폐기될 수밖에 없는 포맷이다. 이 덫에서 빠져나오게 해야 한다."

### 왜 Excel을 못 버리는가 — 심리적 분석

표면적 이유: "내 업무 방식이 다 여기 있어"
실제 이유: **정보 독점을 통한 고용 안정 전략** (의식적이든 무의식적이든)

| 심리 메커니즘 | 내용 |
|---|---|
| 영역 표시 (Territoriality) | "이 파일은 내 것" = 직업적 영역 |
| 불가결성 확보 | "나만 이걸 알아" = 해고 방어 |
| 전환 비용 과대평가 | "바꾸면 다 날아간다" (실제로는 안 그럼) |
| 현상 유지 편향 | 불편해도 익숙한 게 낫다 |

### 절대 하면 안 되는 접근

> "Excel 버리고 Notion 쓰세요."

→ 즉각적 거부 반응. 정체성·안전감 공격으로 받아들임. 상담 즉시 종료.

### 올바른 접근 — 3단계 심리적 전환

```
1단계: 인정 (저항 없애기)
   "Excel 잘 쓰시는 거 맞아요. 그 구조가 다 이유가 있는 겁니다."
   → 신주단지를 부정하지 않는다.

2단계: 확장 (Excel을 더 강하게)
   "이 Excel에 AI를 연결하면 수작업이 사라집니다."
   → Excel을 버리는 게 아니라 업그레이드하는 것으로 포지셔닝.

3단계: 자연스러운 이탈 (결과로 증명)
   자동화 결과를 직접 경험하면 → 자발적으로 더 나은 도구로 이동.
```

### 회사 규모별 도구 제약 — 반드시 먼저 확인

> 대표님 실무 인사이트 (2026-05-25, 삼성전자 재직 경험):
> "Notion 같은 외부 SaaS는 대기업에서 보안상 사용 불가.
>  삼성은 Knox Mail 등 자체 솔루션 사용. 이 점을 반드시 고려해야 한다."

인테이크 시 **첫 번째 질문**으로 추가:

## 맞춤화 자동화 파이프라인 (기술 설계)

개인화가 스케일되려면 LLM 파이프라인이 처리해야 한다.

```

개인화 재작성 (LLM Tier 3) → 부대표 검토 (선택) → 발송

비용:
- Tier 2 분류: Ollama 로컬 (무료)

```

**핵심:** 사람이 직접 쓰는 것처럼 느껴지지만, 파이프라인이 90% 처리.
대표님/부대표님은 검토·발송만.

## 감탄을 설계하는 7가지 원칙

1. **이름을 3번 쓴다** — 오프닝, 핵심 포인트, 마무리
2. **상황을 그대로 반복한다** — "말씀하신 것처럼 엑셀 보고서 작업에서..."
3. **오늘 할 수 있는 것 1가지로 끝낸다** — 막연한 조언 금지
4. **연구를 인용하되 쉽게** — "하버드 연구에서..." (권위) + "쉽게 말하면..." (접근성)
5. **몰랐을 것을 알려준다** — "학원에서는 못 해주는 것"
6. **검색으로 못 찾는 것을 준다** — "구글에 치면 나오는 얘기는 안 합니다"
7. **다음을 기대하게 만든다** — "다음 주에는 000에 대해 말씀드릴게요"
