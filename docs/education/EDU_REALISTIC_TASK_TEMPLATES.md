# EDU 현실형 첫 실습 템플릿

> 작성일: 2026-06-13  
> 목적: `교과서형 과제 금지` 원칙을 실제 구현 가능한 프롬프트/산출물 템플릿으로 구체화한다.

---

## 1. 원칙

첫 실습은 아래를 만족해야 한다.

1. 고객이 지금 수준에서도 바로 시작할 수 있다.
2. 5~10분 안에 눈에 보이는 결과물이 나온다.
3. 결과를 본 뒤 "아, 이런 식으로 쓰는 거구나" 감이 온다.
4. 다음 행동으로 자연스럽게 이어진다.

금지:

- 추상적 질문만 던지는 과제
- 부모가 먼저 AI를 잘 안다는 전제
- 토론/대화만 하라고 하는 과제
- 결과물이 없는 과제

---

## 2. 눈이 뜨이는 경험 정의

아래 중 2개 이상을 만족하면 `눈이 뜨이는 경험`으로 본다.

- 구체적 체크리스트가 나온다
- 바로 복사해 쓸 수 있는 문장이 나온다
- "이건 내가 실제로 오늘 써먹을 수 있겠다"는 감각이 생긴다
- 고객이 걱정하던 상황이 더 구체적인 패턴으로 번역된다
- 다음에 무엇을 할지 명확해진다

---

## 3. 학부모용 템플릿

### 3.1 중등 자녀 AI 숙제 의존 체크포인트

목적:
- 부모가 먼저 "뭘 봐야 하는지" 눈이 뜨이게 한다

권장 툴:
- Claude / Gemini / ChatGPT 중 고객 선택

프롬프트:

```text
중2 자녀를 둔 부모라고 가정하고,
"아이가 AI로 숙제를 베껴 쓰는지 아닌지"를 부모가 집에서 알아차릴 수 있는 체크포인트 7개를 만들어줘.

조건:
1. 교사 말투 말고 부모가 바로 알아들을 말로
2. 너무 공격적이지 않게
3. 실제로 집에서 확인 가능한 행동 신호만
4. 마지막에는 "오늘 바로 해볼 것 2개"를 붙여줘
```

기대 출력:
- 관찰 가능한 행동 신호 7개
- 오늘 할 수 있는 관찰 행동 2개

다음 전환:
- `이 체크포인트 중 우리 집에서 제일 자주 보이는 게 뭔지 표시해보세요`

### 3.2 부모용 AI 설명 문장 만들기

프롬프트:

```text
초등 고학년 아이에게
"AI를 쓰면 안 되는 게 아니라, 잘못 쓰면 생각을 덜 하게 될 수 있다"는 말을
잔소리처럼 안 들리게 설명하는 3가지 버전을 만들어줘.

조건:
- 엄마/아빠가 실제로 말할 수 있는 짧은 문장
- 너무 교훈적이지 않게
- 아이가 반발할 때 다시 말할 문장 1개씩 포함
```

기대 출력:
- 짧은 설명 문장 3세트
- 반발 대응 문장

---

## 4. 성인 자녀 -> 노부모용 템플릿

### 4.1 AI 전화 사기 위험 말투 패턴

프롬프트:

```text
70대 부모님이 AI 전화 사기나 문자 사기를 구분할 때
특히 조심해야 할 말투 패턴 7개를 뽑아줘.

조건:
- 어려운 보안 용어 금지
- 실제 통화/문자에서 바로 알아차릴 표현 중심
- 마지막에 "부모님께 바로 보여줄 1분 요약" 포함
```

기대 출력:
- 말투 패턴 7개
- 부모님께 보여줄 1분 요약

다음 전환:
- `이 1분 요약을 부모님께 보내기 전에 더 쉽게 다듬어볼까요?`

## 5. 직장인용 템플릿

### 5.1 내 업무 자동화 가능 구간 찾기

프롬프트:

```text
내가 하는 일은 마케팅/기획 사무직이고,
반복적으로 시간이 오래 걸리는 일은
보고서 초안 만들기, 회의 요약, 엑셀 정리야.

이 3개 중에서
"AI 초보자도 오늘 가장 먼저 성공 경험을 만들기 좋은 것"
하나를 고르고,
왜 그게 먼저인지 설명한 뒤
지금 바로 복붙해서 써볼 프롬프트 1개를 만들어줘.
```

기대 출력:
- 우선순위 1개
- 이유
- 바로 복붙 가능한 프롬프트

---

## 6. LLM 선택 일치 규칙

같은 실습이라도 최종 카드 문구는 고객이 선택한 LLM 기준으로 바꾼다.

- Claude 선택 고객: Claude Desktop / Claude web 기준
- Gemini 선택 고객: Gemini app / web 기준
- ChatGPT 선택 고객: ChatGPT app / web 기준

금지:

- Claude를 선택한 고객에게 Gemini UI 기준 설명
- Gemini를 선택한 고객에게 Claude Desktop 기준 설치 가이드

---

## 7. 구현 메모

이 문서는 실제 카드 렌더링 전의 원본 템플릿 저장소로 사용한다.

각 템플릿은 아래 메타를 함께 가진다.

- segment
- seeker_role
- target_type
- selected_llm
- difficulty
- expected_output_shape
- next_transition
