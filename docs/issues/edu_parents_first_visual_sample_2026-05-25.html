부모 먼저, 자녀는 나중

    고객용 교육사업 시각화 샘플
  • 대상: 초중등 자녀를 둔 3045세 학부모
  • 목적: "왜 부모가 먼저 AI를 이해해야 하는가"를 1분 안에 납득시키는 orientation 자료
  • 톤: calm, premium, practical, high-readability
ISSUE #002 | PAGE 1

1. 학부모 AI 인식은 높지만 실행 기준은 낮다

  • 많은 부모는 AI가 중요하다는 사실에는 이미 동의한다.
  • 하지만 "우리 집에서는 어디까지 허용할 것인가"라는 운영 기준은 아직 약하다.
  • 따라서 첫 제품은 자녀 교육 팁 모음이 아니라 부모 본인의 AI 사용 기준을 만드는 과정이어야 한다.
  • 이 격차가 Harness 교육 컨설팅의 가장 강한 진입점이다.
ISSUE #002 | PAGE 2

2. 부모 먼저 자녀는 나중

  • 부모가 AI를 직접 써보지 않으면 자녀에게 일관된 기준을 줄 수 없다.
  • 기준이 없는 상태에서 자녀에게 AI만 허용하면, 도구 활용이 아니라 의존이 강화될 수 있다.
  • 따라서 교육 흐름은 부모 체험 가정 기준 수립 자녀 원칙 설계 순서가 맞다.
ISSUE #002 | PAGE 3

3. 첫 주 실행 카드는 가볍고 구체적이어야 한다

  • 교육은 설명에서 끝나면 안 된다.
  • 부모는 긴 리포트보다 "이번 주에 뭘 하면 되는지"를 먼저 원한다.
  • 따라서 follow-up 자산은 문장보다 실행 카드 형태가 적합하다.
ISSUE #002 | PAGE 4

4. 고객 제안용 요약

    이 샘플이 보여주는 것
  • 숫자 차트는 코드 기반으로 정확하게 만든다.
  • 인포그래픽은 설명 구조를 빠르게 납득시키는 데 쓴다.
  • 실행 카드는 학습 내용을 행동으로 연결하는 마지막 장치다.
    • 실제 고객 보고서에 적용할 때
  • 1페이지: 핵심 문제 정의 + readiness gap chart
  • 2페이지: 부모 먼저 구조 인포그래픽
  • 3페이지: 이번 주 실행 카드
  • 4페이지: 프로그램 구조 / 가격 / 다음 행동
    • 제작 원칙
  • 숫자와 축은 코드 원본으로 QA 가능해야 한다.
  • 한글 장문은 생성형 이미지에 박지 않고 후편집한다.
  • 학부모 대상 교육물은 과장된 동기부여 대신 신뢰감과 실행 가능성이 우선이다.
