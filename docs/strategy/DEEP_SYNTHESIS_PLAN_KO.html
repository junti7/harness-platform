1. 배경 및 목적

  • 단순 요약 리포트에서 '데이터 밀도가 높은 프리미엄 인텔리전스'로의 도약을 위해, 수십 개의 팩트 시트를 하나의 권위 있는 내러티브로 결합하는 심층 합성 엔진(Deep Synthesis Engine)이 필요합니다.
  • 또한, 비서실장의 엄격한 품질 승인 프로세스를 시각화하고 자동화하기 위해 OpenClaw 대시보드를 구축합니다. 본 시스템은 MBP에서 개발하고, 24/7 Mac Mini M4에서 안정적으로 운영됩니다.
ISSUE #002 | PAGE 2

2. 핵심 솔루션 및 아키텍처

    2.1 심층 합성 엔진 (Tier 3 업그레이드)
  • 데이터 집계: `extracted_facts`가 포함된 여러 소스를 주제별로 그룹화합니다.
  • Claude 기반 합성: 50개 이상의 팩트 시트(비용, TAM, 성능)를 Claude API에 주입하여 모순을 해결하고 ARK 수준의 심층 리포트를 생성합니다.
  • 자동 차트 연동: `CHART_AUTHORING_PLAYBOOK.md` 규격에 맞춰 차트용 데이터 배열을 자동으로 출력합니다.
    • 2.2 OpenClaw 대시보드 (FastAPI)
  • Fact Viewer: 로컬 모델(Gemma)이 추출한 수치들을 표 형태로 시각화하여 품질을 실시간 검수합니다.
  • Approval Gateway: 비서실장과 부대표가 최종 리포트를 웹에서 미리 보고 '승인/반려'를 결정하는 인터페이스를 제공합니다.

3. 단계별 상세 구현 로직

    Phase 1: 심층 합성 로직 구현
  • `refiner.py`를 재작성하여 단일 기사 요약이 아닌 '배치(Batch) 기반 합성'으로 전환합니다.
    • Phase 2: 대시보드 백엔드 및 UI 개발
  • FastAPI 서버를 구축하고 PostgreSQL DB와 연결하여 팩트 데이터 조회 및 승인 상태 업데이트 API를 개발합니다.
  • Harness 브랜드 아이덴티티에 맞춘 다크 테마의 미니멀 인터페이스를 구축합니다.
    • Phase 4: 파이프라인 승인 잠금(Lock) 적용
  • `run_pipeline.py`가 대시보드 승인을 대기하도록 수정하여, 비서실장 컨펌 없이는 발행(Tier 4)이 절대 불가능하도록 강제합니다.

4. 검증 및 기대 효과

    검증 방법
  • 1. 앞서 수집된 57개의 팩트 데이터를 사용하여 실제 심층 리포트 합성 테스트를 수행합니다.
  • 2. 대시보드에서 '승인' 버튼 클릭 시 Slack으로 PDF가 정확히 발송되는지 전체 워크플로우를 검증합니다.
    • 기대 효과
  • 유료 가치 확보: 단편적인 뉴스가 아닌, 데이터 기반의 거시적 통찰을 제공하여 유료 구독 가치를 극대화합니다.
  • 휴먼 인 더 루프(HITL): 비서실장이 웹 UI를 통해 품질을 최종 통제함으로써 'Zero-Defect'를 실현합니다.
  • 비용 최적화: 로컬 Gemma와 유료 Claude의 역할을 명확히 분리하여 운영 비용을 최소화합니다.
