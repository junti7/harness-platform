## 1. 한 줄 제안

유통 경로에서 소진되지 못한 정상 상품을 낮은 가격에 확보하고, 실제 고객 수요를 먼저 검증한 뒤 소량 판매하여 재고가치를 회복하는 온라인 커머스 사업을 30일 동안 검증한다.

이 사업은 `안 팔리는 물건을 싸게 사는 사업`이 아니다. `판매 근거가 확인된 저평가 재고만 회전시키는 사업`이다.

## 2. 신규 아이템 등록 범위

Harness의 기존 30일 최우선 과제인 weekly issue 4회 발행과 Pretotyping CTR 2% 이상 달성을 침범하지 않는 별도 discovery track으로 등록한다.

- working name: `Harness Value Recovery Mall`
- business type: 재고가치회복형 리커머스 / 소량 셀렉트 커머스
- 초기 고객: 온라인에서 실용 상품을 합리적 가격으로 구매하려는 한국 소비자
- 초기 채널 가설: 스마트스토어 기반 검증, 독립 mall 개발 보류
- 초기 수익 목표: 두 운영자 합계 월 세전 영업이익 200만원
- 목표의 의미: 검증할 경제성 기준. 매출 또는 소득 보장 아님
- Harness 본업과의 시너지 가설: evidence 수집, 실험 설계, agent 운영, 의사결정 게이트를 실제 소매 운영에 적용
- 충돌 위험: 대표·부대표 시간, 현금, 콘텐츠 발행 집중도 분산

현재 production schema의 `business_opportunities` 구현·접속 상태는 이 문서 작성 시 확인하지 않았다. 따라서 DB 등록을 가장하지 않고 이 문서를 canonical opportunity record로 사용한다. 실제 DB 적재는 schema/env/model identity preflight 후 별도 실행한다.

## 3. 왜 검토할 가치가 있는가

2025년 한국 온라인쇼핑 거래액 잠정치는 약 274조 9,869억원이었다. 시장이 크다는 사실은 개별 mall 성공을 보장하지 않지만, 온라인 구매 행동 자체가 충분히 보편적임을 보여준다.[^kostat]

스마트스토어는 2025년 6월 2일 이후 기본 판매수수료를 VAT 포함 3%, 판매자 유입 거래는 1%로 안내한다. 다만 결제 수수료, 광고, 배송, 반품, 솔루션 비용 등 전체 원가와 동일한 숫자는 아니다.[^naver-fee]

따라서 진짜 기회는 `싸게 매입`이 아니라 아래 네 요소의 결합에서 생긴다.

1. 공급가 우위
2. 이미 관찰된 고객 수요
3. 낮은 반품·배송·인증 위험
4. 30~45일 이내 재고회전

## 4. 고객·공급자 문제 가설

### 고객 가설

- 정상 기능 상품을 브랜드 정가보다 저렴하게 사고 싶다.
- 최저가만큼 정품 여부, 상태, 배송, 반품 신뢰를 중요하게 본다.
- 상품이 왜 저렴한지 투명한 설명이 있으면 재고·이월 상품도 구매할 수 있다.

### 공급자 가설

- 제조사, 총판, 폐업·리뉴얼 사업자에게 소량 잔여재고가 있다.
- 기존 도매 채널보다 빠르고 예측 가능한 현금화가 필요하다.
- 정품·매입·인증 증빙과 반품 조건을 제공할 의향이 있다.

### 가장 위험한 가정

- 낮은 매입가 상품에 실제 유료 수요가 존재한다.
- 두 초보 운영자가 상품선정, 상세페이지, CS, 출고를 감당할 수 있다.
- 반품·불량·광고비 반영 후 공헌이익률 25%를 만들 수 있다.
- 소량 매입 조건에서도 공급가 우위가 유지된다.

## 5. 경제성 모델

### 기본식

```text
주문당 공헌이익
= 실결제액
- 상품 매입원가
- 플랫폼·결제 수수료
- 판매자 부담 배송비
- 포장비
- 광고·쿠폰비
- 반품·불량·분실 충당금

월 영업이익
= 주문당 공헌이익 × 월 완료주문 수
- 월 고정비
```

### 목표 역산

두 사람 합계 월 세전 영업이익 200만원을 얻으려면 안전마진 포함 월 영업이익 목표를 250만원으로 둔다.

| 평균 객단가 | 공헌이익률 | 주문당 공헌이익 | 250만원에 필요한 월 주문 | 필요 월매출 |
| ---: | ---: | ---: | ---: | ---: |
| 30,000원 | 20% | 6,000원 | 417건 | 1,251만원 |
| 40,000원 | 25% | 10,000원 | 250건 | 1,000만원 |
| 50,000원 | 30% | 15,000원 | 167건 | 835만원 |

초기 go 기준:

- 광고 전 공헌이익률 30% 이상
- 광고 포함 공헌이익률 25% 이상
- 30일 sell-through 50% 이상, 45일 70% 이상
- 반품률 8% 이하
- 불량·오배송률 2% 이하
- 한 SKU가 운전자금의 15% 초과 금지
- 재고자금 회수기간 45일 이내

위 기준을 하나라도 만족하지 못하면 상품 확대 금지. 월 200만원 목표는 이 기준을 통과한 SKU 묶음이 만들어진 뒤 평가한다.

### 누락비용 포함 3개 scenario

실험 승인안은 아래 항목을 0원으로 두지 않는다.

- 운영자 노동비: 실제 투입시간 × 내부 시간당 원가
- 30/45/60일 미판매 재고의 가격인하·처분손실
- 고객분쟁, chargeback, platform penalty 충당금
- 세무·정산·반품 처리비
- 보관공간과 재포장 비용

| scenario | 판매율·반품·가격 가정 | 의사결정 |
| --- | --- | --- |
| 보수 | 예상 판매량 -30%, 반품·불량 2배, 판매가 -10% | 공헌이익이 음수면 탈락 |
| 기준 | 관찰된 paid test 수치 사용 | 공헌이익률 25% 이상 필요 |
| 낙관 | 판매량 +20%, 반품·불량 -25% | 자본 승인 근거로 단독 사용 금지 |

손익분기 주문수는 `(고정비 + 목표 노동보상) ÷ 주문당 공헌이익`으로 매 실험 전 다시 계산한다. 합계 월 세전 영업이익 200만원은 목표, 250만원은 세금·오차 buffer를 포함한 내부 운영 기준으로 구분한다.

## 6. 초보자용 30일 실행계획

### Phase 0 — 안전 경계 설정, Day 1~3

산출물:

- 역할분담표: 상품·공급 담당 1명, 등록·고객·정산 담당 1명
- 손익계산 sheet
- 금지 카테고리 목록
- 공급자 증빙 checklist
- 반품·불량 비용 기준

초기 금지 카테고리:

- 어린이제품, 전기제품, 배터리 제품
- 식품, 건강기능식품, 화장품, 의료 관련 상품
- 유통기한 상품
- 명품·브랜드 상품 중 정품 유통 증빙 불충분 상품
- 의류·신발처럼 사이즈 반품이 많은 상품
- 대형·파손·설치 상품

제품안전 대상 상품은 인증·신고와 온라인 표시 요건을 확인해야 한다.[^safety]

### Phase 1 — 문제·공급 검증, Day 4~10

- 후보 공급처 20곳 작성
- 최소 10곳과 통화 또는 서면 인터뷰
- SKU 후보 30개 수집
- 매입 증빙, 정품 증빙, KC/표시 대상 여부, 하자율, 반품 조건 확인
- 네이버 쇼핑 및 주요 marketplace에서 최근 가격·리뷰·경쟁 강도 기록
- 상품별 `왜 기존 채널에서 안 팔렸는가` 원인 작성
- 공급처 확인 시점, 당시 가용수량, quote 유효기간 기록
- 매입 전까지 재고가 소진될 수 있음을 기본 가정. 무상 hold가 아니면 확보재고로 표시 금지
- 예약금·독점계약으로 공급재고를 묶는 행위는 `capital_action_approve` 전 금지

통과 조건:

- 증빙 가능한 공급처 3곳 이상
- 조건을 만족하는 SKU 10개 이상
- 공급자 반품 또는 불량 보상 규칙 확인

### Phase 2 — 무매입 Pretotyping, Day 11~17

실제 상품이 준비되지 않은 상태에서 결제를 받지 않는다. 외부 공개 전 Legal, Red Team, QA, 대표 승인이 없으므로 내부 prototype 또는 동의받은 인터뷰만 수행한다.

- SKU별 상품 카드 10개 제작
- 예상 판매가와 `구매 알림 신청` CTA 테스트
- 익명 목표 고객 20명 이상 인터뷰
- 광고비 0원, 대표·부대표 지인에게 결제 요청 금지
- 가격별 구매의향과 불신 이유 기록

통과 조건:

- 참여자 20명 이상
- SKU별 표본 수 명시
- 전체 CTA CTR 2% 이상은 최소 관심 신호
- 상위 SKU 3개가 각각 구매의향 3건 이상 확보
- `싸서 관심`과 `실제 결제 의향` 분리 기록

CTR 2%만으로 구매 수요를 확정하지 않는다. 유료 예약금 또는 실제 주문은 이후 승인된 실험에서만 사용한다.

7일 안에 인터뷰·traffic 표본이 모이지 않아도 실패를 감추기 위해 지인·bot·무관 traffic을 섞지 않는다. Phase 2는 최대 30일까지 연장 가능하며, 표본 미달 상태는 `insufficient_evidence`로 기록한다. 연장 중에도 본업 시간한도는 유지한다.

### Phase 3 — 소량 판매 승인안, Day 18~23

Phase 2 통과 SKU만 대상으로 SKU당 3~5개, 총 재고예산 50만원 이하 실험안을 만든다.

필수 선행:

- `legal_review_approve`
- `red_team_clear`
- `pre_mortem_approve`
- 고객-facing 페이지 `qa_clear`
- 대표 `monetization_experiment_approve`
- 실제 매입 전 별도 `capital_action_approve`

승인 전 상품 매입 또는 판매 개시 금지.

승인 후 paid demand test:

- SKU당 최소 10건, 전체 최소 30건의 실제 주문을 우선 확보
- 예약금 test는 Legal Counsel과 platform 약관 검토가 명시적으로 허용한 경우에만 사용. 가결제, 허위재고, 고의 배송지연 방식 금지
- 예약금을 사용하는 승인된 test는 상품, 예상 배송일, 환불 조건, 판매자 정보를 명확히 표시
- test 종료 후 취소율·환불률과 실제 완료주문을 분리 집계
- `알림 신청`, 인터뷰 발언, CTA click을 paid demand와 합산 금지
- 표본 미달이면 사업성 판정 보류. 긍정 판정 금지
- SKU별 전환율은 표본 수와 함께 95% Wilson confidence interval을 표시. 구간 하한이 내부 최소 전환기준 미만이면 확장 금지

### Phase 4 — 운영 가능성 검증, Day 24~30

승인된 경우에만:

- SKU당 3~5개 판매
- 주문별 실제 원가와 작업시간 기록
- 문의, 취소, 반품, 불량 사유 기록
- 두 운영자의 포장·CS 시간을 분 단위 기록
- 재구매·추천 의향 수집
- CS 최초 응답시간, 출고 lead time, 반품 완료 lead time 기록
- 정산예정액과 실제 입금액 reconciliation
- 영업일·비영업일 CS 당번과 긴급 escalation 연락경로 기록. 주 6시간 cap을 넘기면 신규주문·매입 중단

통과 조건:

- 최소 10건 완료주문
- 광고 포함 공헌이익률 25% 이상
- 반품·불량 기준 이내
- 주문당 운영시간 20분 이하
- 공급처 재주문 조건 유지
- CS 영업일 기준 24시간 이내 응답률 95% 이상
- 취소율 10% 이하, 정산오차율 0.5% 이하
- 노동비 차감 후 순공헌이익/시간이 내부 최저 시간당 원가 이상

10건은 사업 성공 증명이 아니다. 다음 50건 실험으로 갈 최소 운영신호다.

실제 paid test 표본이 전체 30건에 도달하지 않으면 Phase 4 기간을 기계적으로 30일에 끝내지 않는다. `insufficient_evidence`로 보류하고 무매입 또는 승인된 소량 범위에서 표본을 계속 수집한다.

## 7. SKU 선정 scorecard

각 항목 0~5점. 총 40점 중 30점 이상만 후보 유지. `증빙·안전`이 4점 미만이면 총점과 무관하게 탈락.

| 항목 | 질문 |
| --- | --- |
| 수요 | 검색·리뷰·거래 흔적이 있는가? |
| 공급가 | 보수적 판매가 대비 충분히 낮은가? |
| 경쟁 | 최저가 출혈경쟁을 피할 차별점이 있는가? |
| 배송 | 작고 가볍고 파손 위험이 낮은가? |
| 반품 | 사이즈·취향·설치 반품 위험이 낮은가? |
| 증빙·안전 | 정품·매입·인증·표시 증빙이 완전한가? |
| 회전 | 30~45일 안에 팔 근거가 있는가? |
| 콘텐츠 | 사진·설명·bundle로 가치를 전달할 수 있는가? |

## 7A. 재고 손절 규칙

- Day 30: sell-through 50% 미만이면 재주문 중단, 가격 5~10% 조정 test
- Day 45: sell-through 70% 미만이면 원가 이상 회수 가능한 bundle·반송·B2B 재매각 순으로 처분
- Day 60: 남은 재고 전액 손실 가능액을 기록하고 청산. 보유 연장에는 대표 재승인 필요
- SKU당 승인 가능한 최대 손실: 10만원과 실험 재고예산 20% 중 더 작은 금액
- 전체 실험 최대 손실: 20만원. 초과 가능성이 생기면 신규매입 자동 중단
- 공급처 반송 가능 여부와 비용을 매입 전 계약·거래명세에 기록

## 7B. 제품·법무·세무 matrix

Legal Counsel 산출물은 최소 아래 내용을 포함해야 한다. 법률·세무 전문가 자문을 대체하지 않는다.

| 영역 | 실험 전 확인·증빙 |
| --- | --- |
| 사업자·통신판매 | 사업자등록 업종, 통신판매업 신고 필요 여부와 완료 증빙 |
| 소비자 권리 | 청약철회 기간, 반품비 귀책, 환불 시점, 하자 처리정책 |
| 표시·광고 | 판매자 정보, 가격·배송·재고상태, 저렴한 이유, 과장표현 점검 |
| 제품안전 | 제품별 적용 법령, KC/안전확인/공급자적합성, 표시 사진과 인증번호 |
| 정품·유통 | 세금계산서·거래명세서·정품 유통 증빙, lot/serial 보관 |
| 세무·정산 | 일반/간이과세 판단, VAT, 세금계산서·현금영수증, platform 정산대사 |
| 개인정보 | 주문정보 접근권한, 보관기간, 파기, 외부도구 전송 여부 |

증빙 owner는 운영자 A, 고객정책 owner는 운영자 B, 최종 검토 owner는 Legal Counsel Agent다. 증빙은 SKU ID와 연결하고 변경일·검토자·원본 위치를 audit trail에 남긴다.

## 8. 역할과 주간 운영

### 사람

- 대표: opportunity·실험·자본 승인, stop/go 최종 결정
- 부대표: 일반 고객 관점의 상품 신뢰·상세페이지 이해도 검토
- 운영자 A: 공급처, 원가, 증빙, 입고 검수
- 운영자 B: 상품 카드, 고객문의, 주문·반품, 정산

### Harness agents

- Evidence Agent: 수요·가격·공급 증거 수집
- Opportunity Agent: SKU와 사업기회 가설 관리
- Product Planning Agent: 상품군·가격·bundle 설계
- Experiment Agent: Pretotyping과 소량 판매 실험
- CFO Agent: 공헌이익, 현금회전, downside 계산
- Legal Counsel Agent: 표시·광고·전자상거래·인증 검토
- Business Risk Management Agent: 재고·법무·평판 risk register
- QA Agent: 외부 상품페이지의 사실·링크·가격·표시 점검
- Chief of Staff: 필요한 gate 확인 후 대표 보고 승인

## 10. Stop / Go 기준

### 즉시 중단

- 정품·매입·인증 증빙 불완전
- 공급자가 세금계산서·거래명세 등 정상 증빙을 거부
- 반품·불량을 보수적으로 반영하면 공헌이익률 20% 미만
- 노동비·재고처분·분쟁충당금 포함 보수 scenario가 음수
- 구매의향 확보 전 대량 매입 요구
- 대표·부대표 본업 주당 합계 6시간 초과
- 기존 weekly issue 발행 또는 핵심 CTR 실험 지연

### 다음 단계 진행

- 30일 검증 완료
- 소량 실험 공헌이익률 25% 이상
- 최소 10건 완료주문과 운영시간 기록 확보
- 고객이 낮은 가격 외에 신뢰·선택 이유를 설명
- 공급처 재주문 가능
- SKU당 최소 10건, 전체 30건 paid demand 표본 확보

### 확장 검토

- 3개 이상 SKU가 각각 50건 이상 누적 판매
- 2개월 연속 월 영업이익 100만원 이상
- 반품률 8% 이하 유지
- 기존 Harness 우선과제 지연 없음

이후에만 월 영업이익 250만원 규모로 재계획한다. 독립 mall 개발은 marketplace에서 반복수요와 고객획득 경로가 검증된 뒤 검토한다.

## 11. 필요한 대표 결정

현재 요청:

- target_type: `business_opportunity`
- approval_type: `opportunity_approve`
- 의미: 30일 조사·Pretotyping 대상으로 채택
- 실제 돈 집행: 없음

승인되지 않은 것:

- 상품 매입
- 유료 광고
- 외부 판매페이지 발행
- 판매 개시
- 가격·환불정책 확정

### Gate별 필수 evidence contract

| gate | 최소 evidence | 자동 block |
| --- | --- | --- |
| `opportunity_approve` | plan, 위험가정, 무비용 30일 scope | 본업 시간한도 또는 stop rule 없음 |
| `legal_review_approve` | 7B matrix 전 항목, 정책 초안, 증빙 위치 | 미확인 규제·인증 또는 환불정책 누락 |
| plan `red_team_clear` | 서로 다른 허용 모델 2개 output·prompt·model identity, 계획의 안전성·논리 검토 | reviewer 실패, 동일 모델 반복, plan critical/high 미해소 |
| `pre_mortem_approve` | worst case 3개, 확률·손실·탐지·회복 | 최대손실 또는 detection trigger 없음 |
| `qa_clear` | 가격·수수료·링크·표시·모바일 렌더 검증 | 사실·가격·필수표시 불일치 |
| `monetization_experiment_approve` | SKU, 표본, 기간, 환불, 성공·실패 기준, 판매채널 약관 검토 | paid test 표본·환불 조건 또는 channel-policy 확인 없음 |
| `capital_action_approve` | SKU별 매입액, 최대손실, 증빙, 총 cap | 총 50만원 또는 손실 20만원 초과 |

plan Red Team과 experiment Red Team을 혼동하지 않는다. plan `red_team_clear`는 향후 검증 절차가 안전하고 실행 가능한지에 대한 판정이다. 실제 paid demand가 존재한다는 판정이 아니다. paid 결과가 나온 뒤 experiment 결과에 대해 별도 `red_team_clear`를 받아야 확장할 수 있다.

## 11A. 현금흐름 stress test

자본 승인안은 이익과 현금을 분리한다.

- platform 정산이 기준 대비 14일 늦어지는 경우
- VAT·소득세 충당금을 매출 현금과 분리하는 경우
- 주문 30%가 취소되고 반품 환급이 정산보다 먼저 나가는 경우
- 60일 재고를 손절하고 공급처 환불이 없는 경우

보수 scenario에서도 `가용현금 - 향후 30일 확정 지급액`이 0원 미만이 되면 매입을 승인하지 않는다. 사업용 현금, 세금 충당금, 개인 생활비를 분리한다.
