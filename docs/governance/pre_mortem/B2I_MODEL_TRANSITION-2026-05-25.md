# Pre-Mortem: B2C → B2I 사업 모델 전환

- **결정**: Harness 핵심 사업을 B2C 뉴스레터 발행에서 B2I(자기계정 투자) 백엔드로 전환
- **유형**: pre_mortem_approve (high-impact 전략 결정)
- **날짜**: 2026-05-25
- **전체 위험**: **HIGH**
- **권고**: **conditional_go** (조건 충족 후)

## 결론

B2C→B2I 전환은 사업 모델 자체의 근본적 변경이다. 기존 뉴스레터 파이프라인을 내부 투자 신호 생성 백엔드로 재정의하는 것은 타당하나, 발행 공백이 발생할 경우 brand-equity와 구독자 기반이 비가역적으로 손상될 수 있다. 전환 자체에 대한 worst-case 시나리오를 사전에 명확히 인지하고, 복원 가능성을 확보한 상태에서만 진행해야 한다.

**조건부 진행 조건**:
① 발행 완전 중단이 아닌 "뉴스레터 백엔드 전환" — 외부 발행은 보류하되 파이프라인은 유지
② AR-018 red_team_block 7조건 충족 전까지 B2I 엔진 실운용 금지
③ 교육 컨설팅 메인 사업(AI 교육) 진행을 B2I 전환과 병행 — B2I는 내부 백엔드, 교육 컨설팅은 외부 수익 라인 유지
④ 30일 내 paid subscriber 1명 목표는 교육 컨설팅으로 충족 시도, B2I로 대체 불가

## 실패 시나리오

### 시나리오 1: 발행 공백으로 인한 구독자 기반 영구 소멸
- **확률**: 45% (발행 보류 4주 이상 시)
- **최대 손실**: brand-equity 소멸, free subscriber 리스트 사실상 무효화, 재발행 시 재취득 비용 전액 발생
- **복구**: hard_to_reverse (발행 공백 4주 이상이면 사실상 처음부터 재시작)
- **대응**:
  - 발행 "완전 중단"이 아닌 "빈도 조정"으로 운영 (월 1회 이상 유지)
  - 뉴스레터 파이프라인을 B2I 신호 생성 백엔드로 재정의하되 외부 발행 채널은 살려둠
  - 발행 보류 결정 전 부대표 VP 검토 필수
- **조기 신호**: 발행 일정이 3주 이상 연기되거나, Substack/메일리 발행 계획이 아젠다에서 사라지는 경우

### 시나리오 2: B2I 전환이 교육 컨설팅 메인 사업 집중을 희석
- **확률**: 35%
- **최대 손실**: 교육 컨설팅 30일 목표(paid subscriber 1명) 미달성, 초기 수익 창출 시점 지연
- **복구**: reversible (집중도 재조정으로 회복 가능)
- **대응**:
  - B2I는 "내부 백엔드"로 명확히 범위 제한 — 외부 수익 창출 경로는 교육 컨설팅
  - CLAUDE.md 3-Layer 구조 준수: 교육 컨설팅 = 메인 외부 수익, 뉴스레터 = Backend, B2I = 내부 대기
  - 주간 비즈니스 리뷰에서 교육 컨설팅 KPI(Pretotyping CTR, WTP 인터뷰)를 B2I 진척도보다 우선 보고
- **조기 신호**: 주간 리뷰에서 B2I 인프라 작업이 교육 컨설팅 고객 접촉 건수보다 많이 논의되는 경우

### 시나리오 3: "자기계정 투자" 정의 모호화로 자본시장법 경계 침범
- **확률**: 20%
- **최대 손실**: 150,000,000원 이상 (법률 제재 — CAPITAL_ACTION-NA-2026-05-24.md 시나리오 2 참조)
- **복구**: catastrophic
- **대응**:
  - SOUL.md 결정 원칙: "외부 자금 1원도 수탁 금지" 경계를 시스템적으로 강제
  - 향후 구독자 또는 투자자에게 수익 배분하는 구조 절대 불가 (즉시 투자일임업 해당)
  - AR-017 법무 검토 결과를 SOUL.md 결정으로 등록 후 고정
- **조기 신호**: 누군가 "함께 투자하자", "수익 일부 공유" 제안을 하거나, B2I 결과를 외부에 발행/공개하려는 논의가 나오는 경우

### 시나리오 4: LLM 파이프라인 의존으로 "자기 판단" 법적 입증 불가
- **확률**: 30%
- **최대 손실**: 법적 분쟁 시 자기계정 면책 근거 부재
- **복구**: hard_to_reverse
- **대응**:
  - 모든 트레이딩 결정에 CEO 명시적 확인 단계 유지 (human-in-the-loop 원칙)
  - LLM 출력은 "참고 자료"로만 문서화, 최종 판단은 반드시 CEO 결재로 기록
  - 결재 trail: AR tracker + SOUL.md 결정 기록 = 자기 판단 증거
- **조기 신호**: capital_action_approve 없이 자동화 시스템이 주문을 실행하거나, CEO 확인 없이 신호가 주문으로 변환되는 경우

### 시나리오 5: B2I 전환 자체가 실패하고 B2C로 복귀 불가
- **확률**: 15%
- **최대 손실**: 사업 전체 중단, 브랜드 완전 소멸
- **복구**: catastrophic
- **대응**:
  - 전환 결정을 "영구 폐기"가 아닌 "보류(hold)"로 공식화 — B2C AR 상태를 hold로 유지, 복원 절차 명시
  - 뉴스레터 파이프라인 코드베이스 유지 (삭제 금지)
  - B2I 6개월 후 KPI 미달성 시 자동 B2C 재활성화 검토 (KILL_CRITERIA 연동)
- **조기 신호**: B2I 6개월 투자 수익이 LLM API 비용을 초과하지 못하거나, 교육 컨설팅과 B2I 양쪽 모두 수익 0인 상태가 90일 이상 지속
