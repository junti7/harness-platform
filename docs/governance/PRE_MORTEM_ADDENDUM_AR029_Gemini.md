## 시나리오 D — LLM 플랫폼 의존도 (API 정책 변경)

> "핵심 인프라가 단일 LLM 공급업체 정책 변경에 취약하다"

| 항목 | 내용 |
|---|---|
| **시나리오** | Anthropic(Claude) / Google(Gemini) / OpenAI 중 하나 이상이 API 가격 인상, 서비스 약관 변경, 특정 모델 중단을 단행. 또는 한국 서비스 접근 제한. 핵심 파이프라인의 비용 구조가 BEP 이상으로 급등하거나 특정 기능이 중단됨. |
| **발생 확률** | **15~25% (24개월 내)** — AI 인프라 가격 인상은 역사적으로 반복됨. 단, 경쟁 구도가 오히려 가격을 낮추는 요인도 공존. |
| **최대 손실** | LLM 비용이 $100/월 → $200+/월으로 급등 시 BEP 구독자 수 2배로 증가. 또는 핵심 모델(Claude Opus) 중단 시 품질 저하로 구독 이탈. |
| **회복 가능성** | **높음** — Harness는 처음부터 멀티-LLM 전략(Claude + Gemini + Ollama 로컬)을 채택. 단일 공급업체 의존도가 낮음. |
| **Mitigation** | ① **멀티-LLM 아키텍처 유지**: Claude(전략 분석) + Gemini(대량 스캔) + Ollama(Tier 2 저비용) 3-tier 유지 ② **Tier 2 로컬 비중 확대 가능**: Ollama(Mac Mini M4)로 교육 콘텐츠 필터링 대체 시 API 비용 절반 감소 가능 ③ **월 비용 상한 알림**: `DAILY_COST_LIMIT_USD` 변수 활용, 이상 급등 시 즉시 알림 ④ **공급업체 다양화**: Groq(오픈소스), Mistral 등 대안 LLM API 미리 평가 |
| **Detection Trigger** | 월 LLM 비용이 전월 대비 50% 이상 증가 OR 주요 공급업체 가격 변경 공지 발표 → 즉시 CEO 보고 + 비용 구조 재산정 |

## 시나리오 E — 학부모 심리: 불안 ≠ 결제 의향 (Value Proposition 미스매치)

> "AI 의존 불안은 있지만, 해결책으로 '비의존 학습법'에 돈을 내지 않는다"

| 항목 | 내용 |
|---|---|
| **시나리오** | 학부모들은 AI 의존에 불안을 느끼지만, 실제 결제 의향은 "지금 당장 성적을 올려주는 서비스"에 집중됨. "AI 비의존 학습법"이라는 가치 제안이 결제 행동을 유도하지 못하고, 학부모는 오히려 "AI를 더 잘 활용하는 법"(AI 활용 최적화)에 비용을 지불하는 반대 니즈를 보임. |
| **발생 확률** | **30~45%** — 교육 시장에서 "예방적 서비스" 대신 "즉각적 성과 서비스"를 선호하는 패턴이 일반적. WTP 검증 전까지 이 리스크는 실재함. |
| **최대 손실** | Pretotyping 이후 WTP 클릭률 < 2%. 가치 제안 전면 재설계 필요. 3~4주 지연. |
| **회복 가능성** | **높음** — Pretotyping이 조기 발견 기제. 가치 제안을 "AI 비의존" → "AI와 공존하는 학습 설계" 또는 "자녀 AI 활용 현황 정기 진단"으로 재프레이밍 가능. |
| **Mitigation** | ① **Pretotyping 랜딩에서 2가지 메시지 A/B 테스트**: (A) "AI에 의존하지 않는 학습법" vs (B) "AI 시대 자녀 학습 전략 진단" — 어느 메시지가 클릭을 더 많이 유도하는지 측정 ② **WTP 인터뷰 시 가격보다 메시지 먼저**: "얼마를 내겠냐"보다 "이 서비스가 필요하냐"를 먼저 확인 ③ **VP(부대표) 1차 필터**: 학부모 현실 감각으로 랜딩 카피 사전 검토 (VP review 필수) ④ **'불안'이 아닌 '성과'로 재프레이밍**: "AI 시대 자녀 경쟁력 유지"라는 성과 언어로 가치 제안 재구성 검토 |
| **Detection Trigger** | Pretotyping 랜딩 방문자 200명 이상에서 결제 버튼 클릭률 < 2% → 즉시 메시지 A/B 전환 + CEO 보고 |
