## BLOCK ② — "Segment of One" vs 스케일 모순 (가장 중요)

### 문제

EDU_PERSONALIZATION_MODEL.md는 "모든 고객이 '이건 나만을 위한 것'이라고 느껴야 한다"를 핵심으로 내세운다.
동시에 자동화 파이프라인(LLM Tier 2→3)으로 "사람이 90% 처리한 것처럼" 만든다고 설계한다.

이 두 명제는 장기적으로 양립할 수 없다.

### 왜 붕괴하는가

```
초기 (구독자 1~10명):
  템플릿 없음. 실제로 1:1 작성.
  → 진짜 개인화 가능. "감탄" 달성 가능.
  → 비용: 시간 과다 소요, 비확장적.

성장기 (구독자 50~200명):
  효율을 위해 템플릿 기반 LLM 생성 도입.
  → 고객이 비슷한 답변을 받기 시작.
  → "내 얘기 아니잖아?" 인식 → 구독 해지.
  → 또는 진짜 개인화를 유지하면 운영 불가.
```

**핵심 딜레마:** 진짜 개인화는 확장 불가. 확장되면 개인화가 무너진다.

### 해결 방향 제안

이 모순의 해결 없이 "Segment of One"을 마케팅 약속으로 내걸면 고객 기대와 실제 경험 간 갭이 발생한다. 두 가지 선택지:

**옵션 A: "소규모 프리미엄" 포지셔닝으로 재정의**
- "우리는 월 최대 30명만 받습니다" → 진짜 개인화 가능, 프리미엄 가격 정당화
- 스케일 한계 대신 희소성을 강점으로 전환

**옵션 B: 개인화의 정의 재설정**
- "당신 상황을 이해하는 AI + 전문가 리뷰" = 새로운 정의의 개인화
- 완전 수제가 아니라 "AI가 90%, 전문가가 10% 검토" = 투명하게 공개
- 고객에게 "AI 기반 맞춤 상담" 으로 포지셔닝 (기대 조정)

**CEO 결정 필요:** Phase 1에서 "진짜 개인화 (소규모 프리미엄)"로 갈지, "AI 보조 개인화 (투명 공개)"로 갈지.

## BLOCK ⑦ — 리텐션 메커니즘 미정의

### 문제

현재 문서에서 "첫 단추" 이후 구독자가 2, 3개월차에도 돈을 내야 할 이유가 정의되지 않았다.

현재 문서에 있는 것:
- "다음을 기대하게 만든다" (7가지 원칙 中 하나)
- "지속 업데이트 (AI는 계속 변한다)" (공통 콘텐츠 구조)

이것은 구독 유지 이유가 아니라 콘텐츠 생산 원칙이다.

### 왜 위험한가

```
구독 비즈니스의 법칙:
  신규 구독자 유치 비용 > 기존 구독자 유지 비용
  → 리텐션이 낮으면 CAC만 올라가고 LTV는 낮아짐
  → 결국 유료 사업이 아니라 "체험판 배포 사업"이 됨

지금 설계의 문제:
  첫 달: "오 이거 나한테 딱 맞네!" → 구독
  두 번째 달: "저번 달이랑 비슷한데 뭐가 새로운 거지?" → 해지 고민
  세 번째 달: 해지
```

### 해결 방향 제안

**필수 설계 요소 (지금 당장 정의 필요):**
1. "구독자 전용 Q&A 채널" — 질문할 수 있다는 접근권이 리텐션 동기
2. "월 1회 업데이트 브리핑" — AI 교육 트렌드 변화에 따른 처방 업데이트
3. "3개월 추적 상담" — 1개월차 처방 → 3개월 후 어떻게 됐는지 추적
4. "커뮤니티 소속감" — 같은 고민을 가진 사람들의 그룹 (같은 세그먼트)

→ CEO 결정 필요: 리텐션 메커니즘 설계 방향 중 1~2개 선택.

## CONCERN ① — 핵심 사업 전제 타당성

**"부모의 심리적 의지가 아이보다 100배 강하다" (STRATEGIC_SYNTHESIS.md 핵심 전제)**

이 주장 자체는 Trivers(1972) Parental Investment Theory 등 근거 있는 방향이다.

**그러나 비즈니스 전제로 사용할 때 숨겨진 조건이 있다:**

1. **의지 ≠ 지불 의사**: 부모의 의지가 강해도, 새로운 유형의 서비스에 "구독으로 돈을 낸다"는 결정은 별개다. 사교육(학원)은 사회적으로 검증된 지불 패턴이다. Harness는 새로운 카테고리를 만들어야 한다.

2. **강한 의지 → 자체 해결 시도**: 부모 의지가 강할수록 "내가 직접 찾아볼게"라며 무료 유튜브, ChatGPT 직접 사용으로 해결을 시도한다. 이 행동이 Harness의 실질 경쟁자다.

3. **BFI WP 2025-144 "쥐잡기 경주" 함의 재검토**: 이 연구는 "사회적 압박으로 AI를 채택"한다고 했다. 이것은 동시에 "개별 컨설팅보다 주변 따라하기로 의사결정"한다는 의미도 포함된다. Harness가 아닌 "엄마 친구가 하는 학원"으로 흐를 수 있다.

→ 완전 취소 사유는 아니나, "의지가 강하다"를 "돈을 낸다"로 직결하는 논리 연결고리를 명확히 해야 함.

## CONCERN ⑧ — 운영 Bottleneck (대표/부대표 병목)

현재 설계: "LLM이 90% 처리, 대표/부대표가 검토·발송"

**수학:**
```
구독자 100명 × 월 1회 개인 답변 = 월 100건 검토
구독자 200명 × 월 1회 = 월 200건 검토
원포인트 레슨 월 20건 추가

하루 근무 8시간 가정:
200건 ÷ 20일 = 일 10건
10건 × 20분(검토+발송) = 200분 = 3.3시간/일

아직 감당 가능해 보이지만:
→ 구독자 300명 이상이면 검토 업무만으로 풀타임
→ 교육 콘텐츠 생산, 마케팅, 신규 트랙 개발이 불가능
```

→ Phase 2 진입 전 "검토 없이 자동 발송 가능한 티어"와 "반드시 검토가 필요한 티어" 분리 설계 필요.
