## 3. CEO 결정 사항 — 수정된 사업 모델 (검토 대상)

CEO가 2026-05-25 모든 BLOCK에 대한 결정을 내렸습니다. 아래가 **현재 검토 대상인 수정 사업 모델**입니다.

### 3-1. 타겟 (G-1/G-4 해소)

**결정:** 학부모 + 직장인 Phase 1 동시 공략 (파편화 아님)

**근거:**
```
30~45세 초등학생 자녀 둔 워킹맘 = 직장인 pain + 학부모 pain 동시 보유
  직장인 pain: "AI 때문에 내 자리가 불안해"
  학부모 pain: "AI 시대 아이를 어떻게 키워야 하나"
→ 같은 사람의 두 가지 불안 = 오버랩 타겟
```

- 콘텐츠 엔진 하나 (LLM 파이프라인 공유)
- 진입 채널 둘 (맘카페 + 직장인 커뮤니티/LinkedIn)
- CEO 판단: 파편화가 아니라 한 타겟의 두 진입 경로

**Track 구성:**
- Track A (직장인): Step 1 "내 직무 AI 노출도 진단" 무료 → S2~8 분석/전략
- Track B (학부모): Step 1 "우리 아이 AI 의존 위험도" 무료 → S2~8 교육 설계
- 워킹맘 연결: Track A 완료 후 Track B 30% 할인

### 3-2. 가격 모델 (G-3 해소 + BLOCK-2 해소)

**결정:** Step별 마이크로 결제 + lock-in 후 점진 상승 + 데이터 연동 동적 조정

**전체 가격 커브:**

```
S1:무료 → S2:₩990 → S3:₩990 → S4:₩1,490 → S5:₩1,490 → S6:₩1,990 → S7:₩1,990 → S8:₩2,490
[진입]    [마이크로 — 카카오 이모티콘 심리]   [lock-in 진입]         [lock-in 확정]      [완결]

S9(1개월 추적):₩2,990 → S10(2개월):₩2,990 → S11(3개월 완결):₩3,990
[BLOCK-2 해소 — 3개월 추적 코칭, 각 Step별 결제]
```

**패키지:**
- S1~8 풀코스: ₩9,900 (단품 합계 ₩14,890 대비 34% 절감)
- S9~11 추적: ₩7,900 (단품 ₩9,970 대비 21% 절감)
- S1~11 전체: ₩15,900 (단품 ₩24,860 대비 36% 절감)

**데이터 연동 동적 조정:**
- 전환율 ≥ 60%: +10~15% 인상 검토 (A/B 테스트 후)
- 전환율 40~60%: Stay (기본 원칙)
- 전환율 < 40%: -20% 한시 프로모션 또는 콘텐츠 개선 선행

**심리 벤치마크:** 카카오 이모티콘(₩2,200~₩3,300), 네이버 웹툰 미리보기(₩200~₩500/화), 게임 아이템(₩1,100~₩5,500) — 한국인에게 이미 익숙한 마이크로 결제 패턴 탑승

### 3-3. Segment of One 정의 (BLOCK-1 해소)

**결정 (B): AI 90% 자동화 + VP(전문가) 10% 검수, 고객에게 투명 공개**

```
운영 구조:
LLM 파이프라인 (Tier 1~3) → Step 콘텐츠 자동 생성 (90%)
  → 부대표(VP) 학부모/직장인 언어 자연스러움 검수 (10%)
  → 고객 발송

고객에게 공개하는 문구: "AI 기반 분석 + 전문가 검수 서비스"
```

- 스케일 가능 (LLM이 90% 처리)
- VP 검수로 "ChatGPT 복제" 차별화
- 구독자 100명에서도 VP bottleneck 없음 (비동기 검수, 배치 처리)

### 3-4. ChatGPT Wrapper 모트 (G-2 해소)

**결정 (A+C+D): 3중 차별화**

**A — 독점 데이터 축적:**
- 맘카페 정성 분석 데이터 (AI 교육 불안 패턴)
- 한국 학군별 AI 수용도 차이 데이터
- 사용자 Step 반응 데이터 (어떤 진단이 가장 공감됐는가)
- 시간이 지날수록 경쟁자가 복제하기 어려운 자산

**C — VP 학부모 언어 전문성:**
- 부대표가 학부모/직장인 언어 전문가로 성장
- "이 문장이 실제 엄마에게 통하는가" = LLM이 대체할 수 없는 인간 판단
- OJT 프로그램으로 체계적 성장 중

**D — 커뮤니티+데이터 누적 루프:**
- 사용자 Step 완료 반응 → LLM 프롬프트 개선 → 다음 사용자 정확도 향상
- 사용자가 많을수록 서비스 품질 향상 (네트워크 효과)

### 3-5. 고마찰 구조 해소 (G-5 해소)

**결정 (A+B):**

**A — 실행 대행:**
```
Step N 완료 즉시:
  → "이번 주 해야 할 것 3가지" 카드 자동 생성
  → 고객이 따라만 하면 됨 (직접 찾을 필요 없음)
```

**B — 체크리스트 자동화:**
```
Step 완료 후:
  → +3일: "체크해보세요: 1번 해보셨나요?" 카톡 리마인더
  → +7일: "다음 Step 준비됐어요" + 전환 버튼
```

## 5. Codex에게 묻는 구체적 질문

아래 항목들에 대해 **독립적 시각**으로 판단해주세요.

### Q1. 수정된 모델이 Claude/Gemini 이슈를 충분히 해소했는가?

각 BLOCK 항목별로:
- 해소됨 / 부분 해소 / 해소 안됨 중 하나로 평가
- 부분 해소/미해소라면 구체적으로 무엇이 부족한가

### Q2. AI 90% + VP 10% 모델의 VP bottleneck 위험

- 구독자 50명, 100명, 300명 시점에 VP 검수가 실제 병목이 되는가
- 비동기 배치 처리가 가능한가 (Step 완료 → 24시간 내 검수 기준)
- VP가 학부모 언어 전문가로 성장하는 OJT 기간 동안의 품질 공백 관리 방법

### Q3. Step 9~11 (3개월 추적 코칭) LLM 비용 vs 수익 BEP

현재 고정비: LLM API $60~100/월

- Step 9~11 추가 시 LLM 비용 증가분 추정 (각 Step 처리 비용)
- 추적 코칭 Step의 LLM 비용이 수익(₩2,990~₩3,990)을 역전하는 최소 구독자 수
- 3개월 추적이 lock-in으로 작동하기 위한 선행 조건

### Q4. 독점 데이터 확보 경로의 현실성

- A (맘카페 정성 분석): 네이버 ToS §9 자동화 크롤 금지. 수동 수집은 확장성 한계. 실제 독점 데이터로 작동하는가?
- B (학군 AI 수용도 데이터): 이 데이터를 어떻게 수집하는가? 현실적 경로가 있는가?
- D (사용자 반응 데이터): 초기 구독자 0명 시점에는 작동하지 않음. Cold start 문제를 어떻게 해결하는가?

### Q5. 직장인 채널 병행이 초기 집중도를 약화시키는가

- Track A + Track B 동시 운영 시 콘텐츠 생산 부담 (LLM 처리 비용, VP 검수 시간)
- 두 채널을 동시에 광고할 경우 CAC(고객 획득 비용)가 분산되는가
- Gemini의 "타겟 파편화" 지적이 실제로 해소됐는가, 아니면 합리화인가

### Q6. 법적 리스크 (Claude CONCERN-⑥ 미해결)

Claude가 지적했으나 CEO 결정에서 다루지 않은 항목:

- "AI 교육 처방" 서비스의 표시광고법 리스크 (과장 광고)
- "자녀 학습 진단 리포트"의 의료/교육 전문직 유사 행위 여부
- PIPA(개인정보보호법): 음성 온보딩, 자녀 정보 수집 시 동의 설계
- 이 리스크들이 `legal_review_approve` 이전에 사전 검토되었는가
