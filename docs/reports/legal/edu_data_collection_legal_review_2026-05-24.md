## 2. 채널별 세부 검토

### 2.1 RSS 피드 (🟢 즉시 가능)
RSS는 콘텐츠 배포를 위해 설계된 표준 프로토콜입니다. 공개 RSS 엔드포인트를 `feedparser`로 수집하는 것은 사실상 모든 미디어·학술기관이 의도한 방식입니다. 본문 전재(全載)는 금지이며, **인용·요약·출처 표기** 원칙을 지키면 저작권법 위험이 없습니다. 한국: 저작권법 제35조의3(공정이용) 해당 가능성 높음.

### 2.2 arXiv / Semantic Scholar API (🟢 즉시 가능)
두 서비스 모두 API 이용약관에서 **연구목적 비상업적 접근**을 명시 허용합니다. 향후 상업 서비스에 직접 논문 콘텐츠를 포함할 경우 별도 검토 필요. 메타데이터·초록·인용관계 수집은 허용 범위 내입니다.

### 2.3 YouTube 자막 / yt-dlp (🟡 조건부)
YouTube ToS §5B는 "자동화 수단으로 서비스 접근 금지"를 명시합니다. 그러나:
- **실무 현실:** yt-dlp는 전 세계 연구기관·저널리스트·학술기관이 광범위하게 사용 중이며, 법적 선례가 희박합니다(hiQ v. LinkedIn, 2022 공개 데이터 접근 허용 판례 참고).
- **완화 조건:** ① 수집한 자막·메타를 외부 재배포하지 않는다 ② 연구 내부 분석 목적에 한정 ③ 비디오 파일 다운로드 금지(`--skip-download`) ④ 요청 간 rate limiting 적용 ⑤ 대규모 상업화 시 재검토.

→ **판정: 내부 연구 목적·위 조건 하에 착수 가능. 상업 서비스에 자막 원문 포함 시 재검토 필요.**

### 2.4 경쟁사 공개 사이트 (🟡 조건부)
**준수 조건:** ① `robots.txt` 크롤 금지 영역 제외 ② 로그인 없이 공개되는 가격·커리큘럼·후기만 수집 ③ 과도한 요청(DoS 수준) 금지 ④ 수집 결과를 내부 경쟁 분석에만 사용. 이는 판례상 허용된 "공개 정보 비교 조사"에 해당합니다.

### 2.5 네이버 맘카페·블로그 (🔴 별도 조건)
네이버 이용약관은 §9에서 자동화 수집(크롤링·스크래핑)을 **명시 금지**합니다. 한국 저작권법상 게시물은 저작물로 보호받을 수 있습니다. 공개 블로그 글도 저자의 동의 없는 대량 수집은 논란이 될 수 있습니다.

**완화 방안:** ① **공식 Naver Datalab API**(검색 트렌드·키워드 통계) 사용 → 무료, 합법 ② 개별 게시물 인용·요약은 인용 한도 내 수동으로 ③ 대량 자동화 크롤은 외부 법률 전문가 자문 후 결정.

→ **판정: 자동화 대량 크롤 불가(네이버 ToS 위반). Naver Datalab API + 수동 인용으로 대체.**

### 2.6 학부모 설문(1차 데이터, 🟡 조건부)
개인정보보호법(PIPA) 제15조: 개인정보 수집 시 **정보주체 동의** 필수. 완화 조건: ① 동의서 포함한 설문 설계 ② 수집 목적·보유기간 명시 ③ 익명화(이름·연락처 비수집) 시 적용 여부 간소화.

→ **판정: 동의 메커니즘 설계 후 가능. 익명 응답 설문이라면 간소한 고지로 충분.**

### 2.7 Pretotyping 랜딩·광고 (🟡 조건부)
표시광고법: 실제 없는 서비스 또는 효과를 과장한 광고 금지. 완화: ① 랜딩에 "사전 관심 파악용 테스트"임을 명시 ② 효과 수치·사례 없는 경우 명확히 미표기 ③ 결제 전 "베타 참여" 등 pre-launch 표현 사용.

→ **판정: QA qa_clear + 문구 검토 후 가능. 사전 관심 파악 목적 명시 필수.**
